简单工厂模式

简单工厂模式是属于创建型模式，是工厂模式的一种。简单工厂模式是由一个工厂对象决定创建出哪一种产品类的实例。
简单工厂模式是工厂模式家族中最简单实用的模式
简单工厂模式：定义了一个创建对象的类，由这个类来封装实例化对象的行为(代码)
在软件开发中，当我们会用到大量的创建某种、某类或者某批对象时，就会使用到工厂模式.
interface Food{void do();}

class A implements Food{void do(){};}
class B implements Food{void do(){};}

public class FoodFactory {
    public void getFood(String name){
        if ( name.equals("A")) {
            return new A();
        }else( name.equals("B")){
            return new B();
        }
        x.do();
    }
}

class Client{
    public static void main(String[] args) {
        new FoodFactory().getFood("A").do();

工厂方法模式

工厂方法模式：定义了一个创建对象的抽象方法，由子类决定要实例化的类。工厂方法模式将对象的实例化推迟到子类。
interface Food{void do()}

class A implements Food{void do(){}}
class B implements Food{void do(){}}
class C implements Food{void do(){}}
class D implements Food{void do(){}}
//核心，只需传入名字
public abstract class FoodFactory {
    private Food food = null;
    public void eatFlood(String name)){food = getFood(name);food.do();}
    public abstract Food getFood(String name);
}

public class StaticFactory1 extends FoodFactory{
    public food getFood(String name){
        if(name.equals("A")){return new A(); }else{return new B(); }
    }
}

public class StaticFactory2 extends FoodFactory{
    public food getFood(String name){
        if(name.equals("C")){ return new C(); }else{ return new D(); }
    };
}
class Client{
    public static void main(String[] args) {
        StaticFactory2.eatFlood("A");
    }
}

抽象工厂

1) 抽象工厂模式：定义了一个interface用于创建相关或有依赖关系的对象簇，而无需指明具体的类
2) 抽象工厂模式可以将简单工厂模式和工厂方法模式进行整合。
3) 从设计层面看，抽象工厂模式就是对简单工厂模式的改进(或者称为进一步的抽象)。
4) 将工厂抽象成两层，AbsFactory(抽象工厂) 和 具体实现的工厂子类。
程序员可以根据创建对象类型使用对应的工厂子类。这样将单个的简单工厂类变成了工厂簇，更利于代码的维护和扩展
interface Food{void do()}

class A implements Food{void do(){}}
class B implements Food{void do(){}}
class C implements Food{void do(){}}
class D implements Food{void do(){}}

interface AbsFoodFactory {public Food getFood(String name);}   

public class StaticFactory1 implements AbsFoodFactory{
    public static food getFood(String name){
        if(name.equals("A")){return new A() }else{return new B() }
    }
}

public class StaticFactory2 implements AbsFoodFactory{
    public static food getFood(String name){
        if(name.equals("C")){ return new C() }else{ return new D() }
    };
}
//核心。传入工厂和名字
public class FoodFactory {
    AbsFoodFactory factory;
    public FoodFactory(AbsFactory factory) {
        setFactory(factory);
    }
    public void getFood(String name) {
        this.factory.getFood(name).do();
    }
}

class Client{
    public static void main(String[] args) {
        new FoodFactory(new StaticFactory2()).getFood(name);
    }
}

工厂模式小结

将实例化对象的代码提取出来，放到一个类中统一管理和维护，达到和主项目的依赖关系的解耦。从而提高项目的扩展和维护性。
三种工厂模式 (简单工厂模式、工厂方法模式、抽象工厂模式)
设计模式的依赖抽象原则
创建对象实例时，不要直接 new 类, 而是把这个new 类的动作放在一个工厂的方法中，并返回。
变量不要直接持有具体类的引用。
不要让类继承具体类，而是继承抽象类或者是实现interface(接口) 
不要覆盖基类中已经实现的方法。
