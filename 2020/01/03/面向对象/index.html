1. 面向对象概念

  • 面向过程思维方式中更多的体现的是执行者（自己做事情），面向过程性能比面向对象高，因为类调用时需要实例化，开销比较大，比较消耗资源;比如单片机、嵌入式开发、Linux/Unix等一般采用面向过程开发，性能是最重要的因素。
  • 面向对象中更多的体现是指挥者（指挥对象做事情），面向对象性能比面向过程低，易维护、易复用、易扩展，由于面向对象有封装、继承、多态性的特性，可以设计出低耦合的系统
  • 类是对某一类事物的抽象描述，类用于描述多个对象的共同特征，它是对象的模板。
  • 对象用于表示现实中该类事物的个体，它是类的实例

2. 类(引用数据类型)声明

  • 成员方法
    • 形式参数：方法定义中参数列表的变量、实际参数：调用方法时，传入给方法的数值
    • 当传入的数值为基本数据类型（包含String类型），传递的是变量的值
    • 当传入的数值为引用数据类型（String类型除外），传递的是变量的地址
    • 有返回值方法调用:①单独调用②输出调用③赋值调用、无返回值类型的方法调用方式：单独调用
    • 方法重载(overload)同一个类中多个名称相同、参数类型或个数或顺序必须不同的方法（通过JVM来完成同名方法的调用的，通过参数列表来决定调用的是哪一个方法）
      • 方法重载时传入较小的数据类型会自动转成int，传入较大的数据类型的话需要强制转成小类型的
    • 方法重写/覆盖(override)父子类中方法名、参数列表相同的方法，返回值、异常小于等于父类，访问修饰符大于等于父类（
  • 构造方法
    • 成员变量比构造方法先初始化，静态对象比非静态对象先初始化
    • 对象创建时只执行一次的方法、用于初始化类的成员变量，即使不初始化，类成员变量也会有默认值
    • 构造方法名称必须和类型保持一致、想让其他程序无法创建该类的对象，则用private修饰
    • 默认调用已定义的构造方法，没有则自动创建空参构造方法，一个类中允许存在以重载的形式的多个构造方法
    • 如果类中没有构造器，则编译器自动创建一个默认构造器
    • 如果已经定义了一个构造器，编译器不会自动创建默认构造器
    • 构造方法默认第一条语句为super();用来访问父类中的空参数构造方法，进行父类成员的初始化操作
    • super(参数) 与 this(参数) 不能同时在构造方法中存在
    • 构造器不能被继承，因此不能被重写，但可以被重载。
  • 局部代码块{}:定义在方法中的代码块,用来限制变量的作用范围
  • 构造代码块{}:定义在类中方法外的代码块、创建对象时只执行一次，用于对象成员初始化、优先于构造方法执行
  • 静态代码块static{}:定义在类中方法外使用static修饰的代码块、被加载时执行一次，用于给静态变量赋值、类静态成员初始化赋值。优于主方法、构造代码块执行
  • 类的执行顺序：父类静态代码块>子类静态代码块>main方法>父类构造代码块>父类构造方法>子类构造代码块>子类构造方法(取决于main方法放在子类、父类还是无关类中)
  • finalize()方法：一旦垃圾回收器准备好释放对象占用的存储空间，会先调用器finalize()方法，并且在下一次垃圾回收动作发生时，才会真正回收对象占用的内存
    • 有时，若分配内存时采用类类似C语言中的做法，即调用本地方法时需要来释放内存，但不要滥用
  • 垃圾回收器工作原理
    • 存活对象的标记
      • 引用计数：简单缓慢，每个对象含有一个引用计数器，当有引用连接至对象时，引用计数+1，当引用离开作用域或被置为null时，引用计数-1，虽然引用计数开销不大，但这项开销在整个程序的生命周期中持续发生。垃圾回收器会在含有全部对象的列表上遍历，放发现某个对象的引用计数为0时，就释放其占用的空间（通常在计数值变为0时释放对象）。这种方法在对象之间存在循环引用，可能出现对象应该被回收，但其引用计数缺不为0的情况。对垃圾回收器而言，定位这样的交互自引用对象组所需的工作量极大。
      • 从堆栈和静态存储区开始，遍历所有的引用，找到所有活对象，对于每个引用，都最终它所引用的对象，然后是此对象所包含的所有引用，直到源于堆栈换个静态存储区的引用所形成的网络全部被访问为止
    • 垃圾回收技术
      • 停止-复制：暂停程序运行，将所有存活的对象从当前堆复制到另外一个对，没有被复制的全部都是垃圾。当对象被复制到新堆时，它们是一个挨着一个的，所以新堆保持紧凑排列，然后就可以按前述方法简单、直接地分配新空间了
      • 标记-清扫：从堆栈和静态存储区出发，遍历所有的引用，进而找出所有存活的对象。每当找到一个存活对象，就会给对象设一个标记，这个过程不会回收任何对象，只有全部标记完成后，清理动作才会开始。在清理过程中，没有标记的对象将被释放，不会复制，剩下的堆空间是不连续的，要想连续则需要整理剩下的对象
      • 停止-复制

3. 匿名对象

  • 创建对象时，只有创建对象的语句，却没有把对象地址值赋值给某个变量 new 类名([参数]);
  • 只能使用一次、可作为方法接收的参数、方法返回值使用

4. 内部类

  • 将类（内部类）写在其他类（外部类）的内部，内部类可以直接访问外部类的成员，包含私有的成员
    • 使用内部类的优点
      • 一个内部类对象可以访问创建它的外部类对象的内容，包括私有数据！
      • 内部类不为同一包的其他类所见，具有很好的封装性；
      • 内部类有效实现了“多重继承”，优化 java 单继承的缺陷。
      • 匿名内部类可以很方便的定义回调。
    • 内部类有哪些应用场景
      • 一些多算法场合
      • 解决一些非面向对象的语句块。
      • 适当使用内部类，使得代码更加灵活和富有扩展性。
      • 当某个类除了它的外部类，不再被其他的类使用时。
  • 成员内部类：定义在类内部，成员位置上的非静态类。成员内部类可以访问外部类所有的变量和方法。成员内部类依赖于外部类的实例，
  • 静态内部类（nested class）：定义在类内部的静态类，静态内部类可以访问外部类所有的静态变量，而不可访问外部类的非静态变量
  • 局部内部类：定义在方法中的内部类。定义在实例方法中的局部类可以访问外部类的所有变量和方法，定义在静态方法中的局部类只能访问外部类的静态变量和方法
  • 匿名内部类: 没有名字的内部类，必须继承一个抽象类或者实现一个接口。不能定义任何静态成员和静态方法。
    • 当所在的方法的形参需要被匿名内部类使用时，必须声明为 final？
      • 因为生命周期不一致，局部变量直接存储在栈中，当方法执行结束后，非final的局部变量就被销毁。而局部内部类对局部变量的引用依然存在，如果局部内部类要调用局部变量时，就会出错。加了final，可以确保局部内部类使用的变量与外层的局部变量区分开，解决了这个问题
    • 匿名内部类不能是抽象的，它必须要实现继承的类或者实现的接口的所有抽象方法
    • 临时定义某一指定类型的子类并创建刚刚定义的这个子类的对象
    • 匿名内部类是创建某个类型子类对象的快捷方式
  • 内部类的继承，父子类的内部类不能覆盖，只能继承

5. 面向对象的三大特征

  • 封装：把一个对象的属性私有化，同时提供一些可以被外界访问的属性的方法
    • 优点：代码复用、隐藏实现细节、提高安全性
    • 表现：方法、类、private修饰的变量和set/get方法
  • 继承:使用已存在的类的定义作为基础建立新类
    • class 子类 extends 父类 {}
    • 子类自动拥有父类所有非私有属性和方法
    • 一个类不可以继承多个类（单继承），一个类只能有一个直接父类
    • 一个类可以被许多类继承,多层继承是可以的，即一个类的父类可以再去继承另外的父类
    • 子类和父类是一种相对概念、所有的类都直接或者间接的继承了Object类
    • 默认访问子类的[同名]成员变量/方法，不存在则访问父类非私有[同名]成员变量/方法
    • 子类override重写、复写、覆盖父类非私有同名方法，则不理会父类同名方法，必须要保证权限大于等于父类权限（即使在父类构造方法中调用也能自动调子类方法）
    • 默认先执行父类的构造方法，再执行子类的构造方法
    • 子类访问父类中的同名成员变量/方法[构造方法]：super.父类中的成员变量/方法[super()]
    • 优点:代码复用，提供多态前提
    • 缺点:类与类之间的耦合度过高
  • 多态：子类对象(实现类对象)既可以给这个子类(实现类对象)引用变量赋值，又可以给这个子类(实现类对象)的父类(接口)变量赋值
    • 普通类多态：父类 变量名 = new 子类();
    • 抽象类多态：抽象类 变量名 = new 抽象类子类();
    • 接口多态：接口 变量名 = new 接口实现类();
    • 用变量所指向的具体类型和通过该引用变量发出的方法调用在编程时并不确定，而是在程序运行期间才确定，最终多态体现为父类引用变量可以指向子类对象
    • 将一个方法调用同一个方法主体关联起来被称作绑定。若在程序执行前进行绑定（如果有的话，由编译器和连接程序实现），称为前期绑定，在运行时根据对象的类型进行绑定叫做后期绑定或者运行时绑定，java除了static和final方法外，都是后期绑定
    • 方法重载（overload）是编译时的多态性（前绑定），而方法重写（override）是运行时的多态性（后绑定）
    • 优点：配合继承与方法重写提高了代码的复用性与扩展性
    • 实现：1、子父类关系或者类实现接口关系2、方法重写3、父类引用指向子类对象
    • 多态成员方法和变量的特点：方法的运行看右边，其他（编译、运行）都看左边
    • 目的是消除类型与类型之间的耦合
    • 私有、静态的方法、共有的属性不能实现多态（不属于重写）

6. 转型

  • 向上转型:当有子类对象赋值给一个父类引用（多态）
    • 方法传参可以定义为父类，实际参数可以传子类
    • 父类类型 变量名 = (父类类型) 子类类型的变量
    • 好处：隐藏了子类类型，提高了代码的扩展性
    • 弊端：只能使用父类共性的内容，而无法使用子类特有功能
  • 向下转型：一个已经向上转型的子类对象可以使用强制类型转换,将父类引用转为子类引用
    • 如果是直接创建父类对象，是无法向下转型的！
    • 子类类型 变量名 = (子类类型) 父类类型的变量
    • 好处：可以使用子类特有功能
    • 弊端：需要面对具体的子类对象；容易发生ClassCastException类型转换异常。在转换之前必须做类型判断
  • 当不需要面对子类类型时，通过提高扩展性，或者使用父类的功能就能完成相应的操作 当要使用子类特有功能时，就需要使用向下转型

7. 抽象类（基础功能）与接口（扩展功能）

  • 抽象类：被abstract关键字修饰的类是抽象类，定义了抽象方法的类

    • 抽象类可以不定义抽象方法。但抽象方法一定要定义在抽象类中，含有抽象方法的类一定是抽象类
    • 抽象类不可以直接创建对象，原因：调用抽象方法没有意义
    • 只有覆盖了抽象类中所有抽象方法后，其子类才可以创建对象。否则该子类还是一个抽象类
    • 抽象类都是父类，因为不断抽取而来的。
    • 抽象类的存在的意义是为了不让该类创建对象,方法可以直接让子类去使用
    • abstract不可以和private、final、static共存
    • 子类：抽象方法的实现类、抽象类

  • 接口：功能的集合，是比抽象类更为抽象的”类”，只包含了功能声明的特殊类

    • 优点：将功能的声明与实现分离，优化了程序设计，解耦
    • 接口避免了单继承的局限性。父类中定义的事物的基本功能，接口中定义的事物的扩展功能
    • 接口不可以创建对象，可通过多态的方式，由子类来创建对象，接口多态
    • 子类必须覆盖掉接口中所有的抽象方法后，子类才可以实例化。否则子类是一个抽象类
    • 解决了多继承时，当多个父类中有相同功能时，子类调用产生的不确定性。因为没有接口方法体
    • 接口与类关系 class 类 implements 接口1,接口2 {}
    • 接口与接口关系 interface Zi extends Fu1,Fu2,Fu3{}
  • 相同点
    • 都位于继承的顶端,用于被其他类实现或继承
    • 都包含抽象方法,其子类都必须覆写这些抽象方法
    • 都不能直接实例化对象;直到抽象方法都覆盖完了才可以创建对象
  • 不同点
    • 一个类只能继承一个直接父类(可能是抽象类),却可以实现多个接口;(接口弥补了单继承)
    • 抽象类为部分方法提供实现,避免子类重复实现这些方法,提高代码重用性;、接口只能包含抽象方法
    • 抽象类是这个事物中应该具备的你内容, 继承体系是一种 is..a关系
    • 接口是这个事物中的额外内容,继承体系是一种 like..a关系
  • 二者的选用
    • 优先选用接口,尽量少用抽象类
    • 需要定义子类的行为,又要为子类提供共性功能时才选用抽象类

8. 成员变量和局部变量区别

项目 成员变量 局部变量
定义的位置 类中的变量 方法中或者{}语句里面的变量
内存中的位置 堆内存的对象中 栈内存的方法中
生命周期 随着对象的出现而出现在堆中，随着对象的消失而从堆中消失 随着方法的运行而出现在栈中，随着方法的弹栈而消失
初始化 有默认的初始化值 必须手动的给其赋值才可以使用。
作用域 针对整个类有效 只在某个范围内有效。(方法,语句体内)

9. 包的声明与访问

  • 类中声明的包必须与实际class文件所在的文件夹情况相一致，否则，程序运行时会找不到类
  • 类中包的声明：package 包名.包名.包名…; 必须在程序的第一行
  • 包名采用全部小写字母，多层包之间用”.”连接
  • 带有包的类创建对象：包名.类名 变量名 = new包名.类名();
  • 导包：import 包名.类名; import 包名.*;(导入包内所有类)在声明包package后，定义所有类class前
  • 同一个包中的类（同一个文件夹中），或者java.lang包中类时可以省略包名，直接使用该类
  • 类用public修饰，则类名必须与文件名相同。一个文件中只能有一个public修饰的类

11. 常见关键字(小写)

  • this
    • 指向当前对象指针，创建对象的时候就存在
    • 调用本类对象构造方法：this([参数列表]);必须定义在构造方法的第一行，因为初始化动作要最先执行
    • 调用本类对象一般方法：方法名([参数列表])或this.方法名([参数列表])
    • 成员变量名前面加上this.区别成员变量和局部变量
  • super
    • 指向直接父类的指针
    • 调用父类变量：super.变量名
    • 调用父类构造方法：super([实参列表])
    • 调用父类方法：super.方法名([实参列表])
    • 如果我们没写任何的构造方法，编译器提供给我们一个空参数构造方法
    • 如果我们手动给出了构造方法，编译器不会在给我们提供默认的空参数构造方法
    • 子类继承父类中的内容,必须先调用父类的构造方法进行初始化，故子类中所有构造方法有默认的隐式super();故父类的构造方法会先执行
    • 如果默认的隐式super()在父类中没有对应的构造方法，那么必须在构造方法中通过this或者super的形式明确要调用的构造方法
  • final
    • 类、变量或方法不想被继承、修改、重写，可以重载
    • 修饰类、方法不可以被继承，但是可以继承其他类,可以覆盖其他方法时加上final
    • 修饰的变量称为常量，只能赋值一次，必须在创建对象前赋值、没有显式赋值时，多个构造方法都要为其赋值
    • 修饰引用类型的变量值为对象地址值，地址值不能更改、但是地址内的对象属性值可以修改
    • final成员变量
      • 类变量static修饰的变量：直接赋值或者静态代码块中赋值
      • 实例变量：声明变量直接赋初值，非静态初始化块、构造器中赋初值
    • final局部变量
      • 如果final局部变量已经进行了初始化则后面就不能再次进行更改，如果final变量未进行初始化，可以进行赋值，当且仅有一次赋值
    • final方法参数
    • final方法：
      • 确保继承中使方法行为保持不变，而且不会被覆盖
      • 效率，编译器将针对该方法的所有调用转为内嵌调用，跳过插入程序代码的方式而执行方法调用机制（将参数压入栈，跳至方法代码处执行，然后跳回并清理栈中的参数，处理返回值），并且以方法体中的实际代码的副本替代方法调用，消除方法的开销
  • static
    • 无需创建对象，独立于对象存在，多个对象共享
    • 类的加载的时候只执行一次（性能优化），不能使用this/super
    • 静态修饰的内容（方法、变量）存于静态区,同一个类中，静态成员只能访问静态成员，但非静态成员能访问静态成员
    • 调用：类名.静态成员变量名\类名.静态成员方法名(参数)
