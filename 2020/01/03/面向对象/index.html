面向对象的三大特征

面向对象概念

  • 面向过程思维方式中更多的体现的是执行者（自己做事情），面向对象中更多的体现是指挥者（指挥对象做事情）
  • 类是对某一类事物的抽象描述，而对象用于表示现实中该类事物的个体
  • 类用于描述多个对象的共同特征，它是对象的模板。对象用于描述现实中的个体，它是类的实例

类(引用数据类型)

分类 说明
类的定义 [public] class 类名{访问修饰符 数据类型 属性名(s);成员方法;构造方法;内部类;}
成员方法 访问修饰符 返回值类型/void 方法名([参数类型 参数名(s)]//参数列表){执行语句……… [return 返回值;]}
形式参数：方法定义中参数列表的变量、实际参数：调用方法时，传入给方法的数值
当传入的数值为基本数据类型（包含String类型），复制的是变量的值，形式参数的改变对实际参数不影响
当传入的数值为引用数据类型（String类型除外），复制的是变量的地址，形式参数的改变对实际参数有影响
有返回值方法调用:①单独调用②输出调用③赋值调用、无返回值类型的方法调用方式：单独调用
方法重载（overload） Java允许在一个类中定义多个名称相同的方法，但是参数的类型或个数或顺序必须不同
是通过JVM来完成同名方法的调用的，通过参数列表来决定调用的是哪一个方法
构造方法 修饰符 类名(参数列表){} 对象创建时只执行一次的方法、用于初始化类的成员变量
构造方法名称必须和类型保持一致、想让其他程序无法创建该类的对象，则用private修饰
默认调用已定义的构造方法，没有则自动创建空参构造方法，一个类中允许存在以重载的形式的多个构造方法
构造方法默认第一条语句为super();用来访问父类中的空参数构造方法，进行父类成员的初始化操作
super(参数) 与 this(参数) 不能同时在构造方法中存在
局部代码块{} 定义在方法中的代码块,用来限制变量的作用范围
构造代码块{} 定义在类中方法外的代码块、优先于构造方法执行，用于执行所有对象中成员的初始化赋值
每创建一个对象均会执行一次构造代码块
静态代码块static{} 定义在类中方法外使用static修饰的代码块、优于主方法、构造代码块执行，当以任意形式第一次使用到该类时执行
该类不管创建多少对象，静态代码块只执行一次、可用于给静态变量赋值、类静态成员初始化赋值
匿名对象 创建对象时，只有创建对象的语句，却没有把对象地址值赋值给某个变量new 类名([参数]);
只能使用一次、可作为方法接收的参数、方法返回值使用
内部类 将类（内部类）写在其他类（外部类）的内部，写在其他类的成员位置和局部位置
内部类可以直接访问外部类的成员，包含私有的成员
成员内部类：class 外部类 { 修饰符 class 内部类 {//其他代码}}
外部类名.内部类名 变量名 = new 外部类名().new 内部类名();
局部内部类：class 外部类 {修饰符 返回值类型 方法名(参数) {class 内部类 {//方法}new 内部类().内部类中的方法();}}
在外部类方法中，创建内部类对象，进行访问

面向对象的三大特征

特征 项目 说明
封装 优点 代码复用、隐藏实现细节、提高安全性
表现 方法、类、private修饰的变量和set/get方法
继承 特点 子类自动拥有父类所有非私有属性和方法
一个类不可以继承多个类（单继承），一个类只能有一个直接父类
一个类可以被许多类继承,多层继承是可以的，即一个类的父类可以再去继承另外的父类
子类和父类是一种相对概念、所有的类都直接或者间接的继承了Object类
默认访问子类的[同名]成员变量/方法，不存在则访问父类非私有[同名]成员变量/方法
子类override重写、复写、覆盖父类非私有同名方法，则不理会父类同名方法，必须要保证权限大于等于父类权限
默认先执行父类的构造方法，再执行子类的构造方法
子类访问父类中的同名成员变量/方法[构造方法]：super.父类中的成员变量/方法[super()]
表现 class 子类 extends 父类 {}
优点 代码复用，提供多态前提
缺点 类与类之间的耦合度过高
多态 定义 子类对象(实现类对象)既可以给这个子类(实现类对象)引用变量赋值，又可以给这个子类(实现类对象)的父类(接口)变量赋值
最终多态体现为父类引用变量可以指向子类对象
普通类多态：父类 变量名 = new 子类();
抽象类多态：抽象类 变量名 = new 抽象类子类();
接口多态：接口 变量名 = new 接口实现类();
优点 配合继承与方法重写提高了代码的复用性与扩展性
前提 1、子父类关系或者类实现接口关系2、方法重写3、父类引用指向子类对象
特点 成员方法和变量的特点：方法的运行看右边，其他（编译、运行）都看左边
转型 向上转型:当有子类对象赋值给一个父类引用（多态）
好处：隐藏了子类类型，提高了代码的扩展性
弊端：只能使用父类共性的内容，而无法使用子类特有功能
向下转型：一个已经向上转型的子类对象可以使用强制类型转换,将父类引用转为子类引用
如果是直接创建父类对象，是无法向下转型的！
子类类型 变量名 = (子类类型) 父类类型的变量;
好处：可以使用子类特有功能
弊端：需要面对具体的子类对象；容易发生ClassCastException类型转换异常。在转换之前必须做类型判断
选用 当不需要面对子类类型时，通过提高扩展性，或者使用父类的功能就能完成相应的操作
当要使用子类特有功能时，就需要使用向下转型

抽象类（基础功能）与接口（扩展功能）

分类 项目 说明
抽象类 定义 被abstract关键字修饰的类是抽象类，定义了抽象方法的类
abstract class 类名 {[public abstract 返回值类型 方法名(参数);(s)]}
注意事项 抽象类可以不定义抽象方法。但抽象方法一定要定义在抽象类中
抽象类不可以直接创建对象，原因：调用抽象方法没有意义。
只有覆盖了抽象类中所有抽象方法后，其子类才可以创建对象。否则该子类还是一个抽象类
抽象类都是父类，因为不断抽取而来的。
抽象类的存在的意义是为了不让该类创建对象,方法可以直接让子类去使用
abstract不可以和private、final、static共存
子类 抽象方法的实现类、抽象类
接口 定义 功能的集合，是比抽象类更为抽象的”类”，只包含了功能声明的特殊类
public interface 接口名{[public static final] A_B=值;//常量，接口变量，不可被修改
[public abstract] 返回值类型 方法名(数据类型 变量名(s));//抽象方法}
优点 将功能的声明与实现分离，优化了程序设计，解耦
特点 定义接口所在的文件仍为.java文件，编译后仍然会产生.class文件
接口只描述所应该具备的方法，并没有具体实现，具体的实现由接口的实现类来完成
接口不可以创建对象，可通过多态的方式，由子类来创建对象，接口多态
子类必须覆盖掉接口中所有的抽象方法后，子类才可以实例化。否则子类是一个抽象类
接口与类关系 class 类 implements 接口1,接口2 {}
解决了多继承时，当多个父类中有相同功能时，子类调用产生的不确定性。因为没有接口方法体
interface Zi extends Fu1,Fu2,Fu3{}
class Zi extends Fu implements Inter{}
接口避免了单继承的局限性。父类中定义的事物的基本功能，接口中定义的事物的扩展功能
二者异同 相同点 都位于继承的顶端,用于被其他类实现或继承;
都包含抽象方法,其子类都必须覆写这些抽象方法;
都不能直接实例化对象;直到抽象方法都覆盖完了才可以创建对象
不同点 一个类只能继承一个直接父类(可能是抽象类),却可以实现多个接口;(接口弥补了单继承)
抽象类为部分方法提供实现,避免子类重复实现这些方法,提高代码重用性;、接口只能包含抽象方法;
抽象类是这个事物中应该具备的你内容, 继承体系是一种 is..a关系
接口是这个事物中的额外内容,继承体系是一种 like..a关系
二者的选用 优先选用接口,尽量少用抽象类;
需要定义子类的行为,又要为子类提供共性功能时才选用抽象类;

成员变量和局部变量区别

项目 成员变量 局部变量
定义的位置 类中的变量 方法中或者{}语句里面的变量
内存中的位置 堆内存的对象中 栈内存的方法中
声明周期 随着对象的出现而出现在堆中，随着对象的消失而从堆中消失 随着方法的运行而出现在栈中，随着方法的弹栈而消失
初始化 有默认的初始化值 必须手动的给其赋值才可以使用。

包的声明与访问

  • 类中声明的包必须与实际class文件所在的文件夹情况相一致，否则，程序运行时会找不到类
  • 类中包的声明：package 包名.包名.包名…;
  • 包名采用全部小写字母，多层包之间用”.”连接
  • 带有包的类创建对象：包名.类名 变量名 = new包名.类名();
  • 导包：import 包名.类名; import 包名.*;(导入包内所有类)在声明包package后，定义所有类class前
  • 同一个包中的类（同一个文件夹中），或者java.lang包中类时可以省略包名，直接使用该类
  • 类用public修饰，则类名必须与文件名相同。一个文件中只能有一个public修饰的类

访问修饰符

public protected default private
同一个类中 * * * *
同包的子类和无关类 * * *
不同包的子类 * *
不同包的无关类 *

常见关键字(小写)

关键字 说明
this 成员变量名前面加上this.区别成员变量和局部变量
调用本类对象构造方法：this([参数列表]);必须定义在构造方法的第一行，因为初始化动作要最先执行
调用本类对象一般方法：方法名([参数列表])或this.方法名([参数列表])
this代表当前对象，创建对象的时候就存在
super 表示当前对象中包含的父类对象空间的引用
调用父类构造方法：super([实参列表]);
调用父类方法：super.方法名([实参列表]);
子类继承父类中的内容,必须先调用父类的构造方法进行初始化
故子类中所有构造方法有默认的隐式super();故父类的构造方法会先执行
如果默认的隐式super()在父类中没有对应的构造方法
那么必须在构造方法中通过this或者super的形式明确要调用的构造方法
如果我们手动给出了构造方法，编译器不会在给我们提供默认的空参数构造方法
如果我们没写任何的构造方法，编译器提供给我们一个空参数构造方法
final 类、变量或方法不想被继承、修改、重写
修饰类不可以被继承，但是可以继承其他类
修饰的变量称为常量，只能赋值一次，必须在创建对象前赋值、没有显式赋值时，多个构造方法都要为其赋值
修饰引用类型的变量值为对象地址值，地址值不能更改、但是地址内的对象属性值可以修改
修饰方法不可以被覆盖,可以继承过来使用，可以覆盖其他方法时加上final
static 不创建对象，多个对象共享同一个static成员变量和方法
类名.静态成员变量名\类名.静态成员方法名(参数)
静态常量public static final 数据类型 AA_BB = 值;
静态内容是优先于对象存在，随类的加载而加载，不能使用this/super
静态修饰的内容存于静态区,同一个类中，静态成员只能访问静态成员
main方法为静态方法仅为程序执行入口，不属于任何一个对象，可以定义在任意类中
instanceof 判断某个对象是否属于某种数据类型,boolean b = 对象 instanceof 数据类型;
