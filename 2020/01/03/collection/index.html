集合与数组的异同

  • 数组的长度是固定的。集合的长度是可变的。集合中存储的元素必须是引用类型数据
  • 集合中存储其实都是对象的地址
  • 集合中可以存储基本数值吗？可以通过基本类型包装类，自动装箱操作存储
  • 没指定<>时，元素存储时自动提升为Object。取出时要使用元素的特有内容，必须向下转型
  • 如果集合中存放的是多个对象，这时进行向下转型会发生类型转换异常
  • Iterator接口使用<>控制迭代元素的类型就不需要强转了.获取到的元素直接就是元素类型

数据存储常用的结构

  • 堆栈:先进后出、栈的入口、出口的都是栈的顶端位置、压栈、弹栈
  • 队列:先进先出、队列的入口、出口各占一侧
  • 数组:查找元素快：通过索引快速访问指定位置的元素
  • 增删元素慢：需要创建新数组，将指定新元素存储在指定索引位置，再把原数组元素根据索引，(不)复制到新数组对应索引的位置
  • 链表:多个节点之间，通过地址进行连接
  • 查找元素慢：想查找某个元素，需要通过连接的节点，依次向后查找指定元素
  • 增删元素快：只需要修改连接下个元素的地址即可

集合框架继承体系

java.lang.Object：所有java类的超类

java.util.AbstractCollection (implements java.util.Collection)

  • void add(Object obj) 添加对象
  • boolean remove(Object o) 移除集合中指定的元素
  • void clear() 清空集合中的所有元素、集合容器本身依然存在
  • Object[] toArray() 集合中的元素,转成一个对象数组中的元素并返回
  • boolean contains(Object o) 判断对象是否存在于集合中,对象存在返回true

java.util.AbstractList (implements java.util.List)

  • void add(int index,E element) 在列表中插入指定元素
  • Object remove(int index) 移除列表中指定位置的元素, 返回的是被移除的元素
  • Object set(int index, Object element) 指定元素替换集合中指定位置元素,返回更新前元素
  • Object get(int index) 返回集合中指定位置的元素

java.util.AbstractSequentialList

java.util.LinkedList

  • 链表、队列结构
  • void addFirst(Element e); 将指定元素插入此表开头
  • void addLast(Element e); 将指定元素插入此表末尾
  • Element removeFirst(); 移除并返回此列表的第一个元素
  • Element removeLast(); 移除并返回此列表的最后一个元素
  • void push(Element e); 将元素推入此列表所表示的堆栈
  • Element pop() 从此列表堆栈出弹出一个元素
  • Element getFirst(); 返回此表的第一个元素
  • Element getLast(); 返回此表的最后一个元素
  • Boolean isEmpty() 如果列表不包含元素，则返回

java.util.ArrayList

  • 数组结构、存取有序
  • int clear()返回集合大小

java.util.Vector

  • 数组、枚举结构
  • void addElement(Element Obj) 将指定的组件添加到此向量的末尾
  • Element elementAt(int index) 返回指定索引处的组件
  • Enumeration elements() 返回此向量组件的枚举（遍历）
  • boolean hasMoreElements() 测试此枚举是否包含更多的元素
  • Element nextElement() 返回此枚举的下一个元素

java.util.AbstractSet (implements java.util.List)

  • boolean add(Object element) 将指定的元素，添加到该集合中的末尾
  • void add(int index,E element) 在列表中插入指定元素
  • Object remove(int index) 移除列表中指定位置的元素, 返回的是被移除的元素
  • Object remove(Object element) 移除列表中第一次出现的元素, 返回的是被移除的元素
  • Object set(int index, Object element) 指定元素替换集合中指定位置元素,返回更新前元素
  • Object get(int index) 返回集合中指定位置的元素

java.util.HashSet(implements java.util.List)

  • （数组机制）存取无序

java.util.LinkedHashSet

  • 链表和哈希表组合，元素存取有序，不重复

java.util.AbstractMap<K,V> (implements java.util.Map<K,V>)

  • Map中的集合不能包含重复的键，值可以重复；每个键只能对应一个值。
  • V put(K key, V value)把指定的键值添加到Map集合中返回null或替换前的值
  • V remove(Object key)删除指定键对应的值，返回被删除元素的值
  • V get(Object key) 根据指定的键，在Map集合中获取对应的值
  • Set keySet() 获取Map集合中所有的键，存储到Set集合中
  • Set<Map.Entry<K,V>> entrySet() 获取到Map集合中所有的键值对对象的集合(Set集合)
  • K getKey() 返回与此项对应的键
  • V getValue()返回与此项对应的值

java.util.HashMap

  • 哈希表结构,元素存取无序,值重复,键重复则覆盖,要重写键的hashCode、equals方法

java.util.LinkedHashMap

  • 元素的存取有序,哈希表结构,值重复,键重复则覆盖,要重写键的hashCode,equals方法

java.util.Collections

  • static void sort(List list) 升序排序
  • static void shuffle(List list) 集合中的元素存储位置随机打乱

集合contains方法判断集合元素唯一的原理

  • List:默认根据equals方法中的==判断地址值是否一样
  • 如果想根据内容判断是否为重复元素，需要重写元素的equals方法
  • Set、Map:默认先根据hashCode方法的hash值是否一样，然后根据equals方法中的==判断地址值是否一样
  • 如果自定义类型想根据内容判断是否为重复元素，需要重写元素的hashcode和equals方法
  • Java提供的包装类一般都重写了hashcode和equals方法，不需再重写

java.lang.Iterable

  • ArrayList迭代
  • Vector枚举迭代
  • Map迭代
  • 增强for
  • 好处: 代码少了,方便对容器遍历
  • 弊端: 没有索引,不能操作容器里面的元素
  • 并发修改异常
  • 在迭代的过程中使用了集合的方法对集合的元素进行操作会引发并发修改异常
  • 解决方法：①使用迭代器的方法操作元素②使用for循环操作元素

泛型

  • 泛型概念
    • 指明了集合中存储数据的类型 <数据类型>(安全简单)
    • 在编译的时候检查类型安全,避免了迭代时类型强转的麻烦
  • 泛型声明
    • 泛型类 修饰符 class 类名<代表泛型的变量> {}
    • 泛型方法 修饰符 <代表泛型的变量> 返回值类型 方法名(参数){} public T[] toArray(T[] a){}
    • 泛型接口 public interface List {abstract boolean add(E e);}
  • 通配符和泛型的限定
    • Iterator<?> it = coll.iterator()泛型的通配,匹配所有的数据类型
    • <? extends Employee> 限制的是父类, 上限限定, 可以传递Employee,传递他的子类对象
    • <? super Employee> 限制的是子类, 下限限定, 可以传递Employee,传递他的父类对象
  • Java伪泛型
    • 泛型只在编译时存在,编译后就被擦除,在编译之前我们就可以限制集合的类型,起到作用
    • 例如:ArrayList al=new ArrayList();编译后:ArrayList al=new ArrayList();
