MyBatis、JDBC、Statement与PreparedStatement区别？#{}、${}区别？ √模糊查询 like 语句该怎么写？

半ORM（对象关系映射）框架，封装了JDBC，开发时只关注 SQL 语句本身，不需要去处理加载驱动、创建连接、创建 statement 等繁杂的过程。可以严格控制 sql 执行性能，灵活度高。
sql语句编写工作量大，要求高，数据库移植性差

Java DataBase Connectivity用于执行SQL语句的Java API接口

demo
//1.注册驱动
Class.forName(“com.mysql.jdbc.Driver”);
//2.获得链接
connection conn = DirverManger.getConnection(jdbc:mysql://ip:port/test,，密码);
String sql = "select \* from t where a = ?";
PreparedStatement psmt = conn.prepareStatement(sql);//3.获得预处理对象语句执行平台
//void setXxx(int index, Xxx xx); --设置实际参数
psmt.setString(1,'张三');//4.设置实际参数
//int executeUpdate(); --执行insert update delete语句.
//ResultSet executeQuery(); --执行select语句.
//boolean execute(); --执行select返回true 其他的语句返回false
ResultSet rs = psmt.executeQuery();//5.执行sql
while( rs.next() ){//6.处理结果集
	Object getObject(int index/String name) 获得任意对象
	String getString(int index/String name) 获得字符串
	int getInt(int index/String name) 获得整形
	double getDouble(int index/String name) 获得双精度浮点型
}
rs.close();//7.释放所有资源
stmt.close();
con.close();

用途：执行静态 SQL；执行动态 SQL，支持参数化
性能：每次都解析和编译，性能差；预编译，复用SQL模板，高性能
安全性：容易受到SQL注入攻击；防止SQL注入
可读性和维护性：较差，难维护 较好，参数化设计清晰
批处理：不适合，支持批量操作

#{}是占位符，预编译处理；${}是拼接符，字符串替换，没有~
#{}传入字符串参数，替换为?号，调用PreparedStatement的set方法赋值
#{}能防止SQL注入，提高系统安全性；${}不能~
#{}的变量替换是在DBMS中；${} 的~外
#{}自动进行java类型和jdbc类型转换。自动加上''。${}不进行java类型和jdbc类型转换， 不加'' 单条SQL不超过语句的最大限制max_allowed_packet（1m）

CONCAT('%',#{question},'%')或者"%"#{question}"%"

Mybatis延迟加载？原理？执行一对一、一对多的关联查询？

配置文件配置lazyLoadingEnabled=true
使用CGLIB创建目标对象的代理对象，当调用目标方法时，进入拦截器方法，比如调用 a.getB().getName()，拦截器 invoke()方法发现 a.getB()是 null则发送事先保存好的查询关联 B 对象的 sql查询B，然后调用 a.setB(b)，于是 a 的对象 b 属性就有值了，接着完成 a.getB().getName()方法的调用
Hibernate同理

association一对一关联对象，collection一对多查询关联集合对象的延迟加载。

MyBatis事务管理形式？

使用JDBC的事务管理机制。利用java.sql.Connection对象
使用MANAGED的事务管理机制。让容器如WebLogic、JBOSS

MyBatis一级缓存和二级缓存？

不推荐，因为分布式环境下缓存基于本地，会有脏数据，不如直接使用Redis、Memcached等分布式缓存
一级缓存（默认打开）：基于PerpetualCache没有容量限定的HashMap缓存，存储作用域为sqlSession或者statement，建议设为Statement。配置：<setting name="localCacheScope" value="SESSION|STATEMENT"/>

二级缓存：基于~，~Mapper(Namespace)，可自定义存储源Ehcache。配置：<setting name="cacheEnabled" value="true"/>在映射XML中配置cache或者cache-ref
当一级缓存Session/二级缓存Namespaces进行了C/U/D操作后，默认该作用域下所有select的缓存将被清楚

MyBatis执行顺序√

读取全局配置文件 mybatis-config.xml：获取运行环境和数据库连接信息。
加载映射文件 Mapper.xml：加载操作数据库的 SQL 映射文件，将 SQL 配置与 MyBatis 的运行环境关联。
基于上面的配置文件生成Configuration，为每个 SQL 映射生成对应的 MappedStatement 对象，并存入 Configuration。
构建会话工厂：通过配置文件信息创建 SqlSessionFactory 对象。SqlSessionFactoryBuilder 通过 Configuration 对象生成 SqlSessionFactory
创建 SqlSession 对象：由会话工厂创建 SqlSession 对象，用于执行 SQL 操作。
在 Executor 接口的执行方法中，包含 MappedStatement 参数：MyBatis 将 Mapper.xml 中的每个 SQL 定义为一个 MappedStatement 对象，用于存储 SQL 的 ID、参数等信息。
MyBatis 底层通过 Executor 接口执行 SQL 并维护缓存：根据 SqlSession 传递的参数动态生成 SQL，调用数据库并处理缓存逻辑。
输入参数映射：在执行 SQL 前，通过 MappedStatement 对象将输入参数映射到 SQL 语句中。
输出结果映射：在 SQL 执行完成后，通过 MappedStatement 对象将结果集映射为 Java 对象。

MyBatis 中 SQLSession 流程

用户程序调用 MyBatis 接口层 API（即 Mapper 接口中的方法）。用户通过调用 Mapper 接口的方法来触发 SQL 执行。
SqlSession 通过调用 API 的 StatementID 找到对应的 MappedStatement 对象。根据调用的 StatementID，SqlSession 会找到对应的 MappedStatement 对象，该对象包含 SQL 的配置信息。
通过 Executor（负责动态 SQL 的生成和查询缓存的维护）将 MappedStatement 对象进行解析。Executor 负责执行 SQL，并处理 SQL 执行过程中的各种操作，如动态 SQL 生成和缓存管理。
使用 ParameterHandler 填充参数，使用 StatementHandler 绑定参数。在 SQL 执行前，ParameterHandler 会填充参数，StatementHandler 会将参数绑定到 SQL 语句中。
SQL 参数转化、动态 SQL 拼接，生成 JDBC Statement 对象。如果有动态 SQL，MyBatis 会在此阶段进行拼接，并生成 JDBC 的 Statement 对象。
JDBC 执行 SQL 语句。Executor 通过 Statement 执行 SQL。
使用 ResultSetHandler 将返回结果转化成 HashMap、JavaBean 等存储结构并返回。执行完 SQL 后，ResultSetHandler 将查询结果从 ResultSet 转换为 Java 对象（如 HashMap 或 POJO）。
借助 MappedStatement 中的结果映射关系。ResultSetHandler 使用 MappedStatement 中的结果映射关系将数据库查询结果映射到 Java 对象。
关闭 SqlSession 会话。执行完 SQL 后，SqlSession 会关闭会话，释放资源。

动态SQL概念，作用，执行原理？

以XML标签的形式编写动态SQL ，完成逻辑判断和动态拼接SQL的功能。
动态SQL标签：<if />、<choose />、<when />、<otherwise />、<trim />、<where />、<set />、<foreach />、<bind /> 。
原理：使用OGNL的表达式，从SQL参数对象中计算表达式的值

Mapper接口与XML对应关系？Mapper接口方法能重载吗？

接口的全限名、方法名、方法参数映射文件namespace值、MappedStatement的id值、SQL的参数
不能重载，因为是全限名+方法名的保存和寻找策略。

原理：select、insert、update、delete标签都解析为MappedStatement对象。Mapper接口的实现类通过使用JDK动态代理自动生成代理对象Proxy时会拦截接口方法，根据接口全限名+方法名拼接字符串作为key值，唯一定位一个对应的MappedStatement执行SQL

Mybatis是否可以映射Enum枚举类？

EnumTypeHandler基于Enum.name枚举名称(String)。默认。
EnumOrdinalTypeHandler基于Enum.ordinal枚举数值(int)。
可通过 <setting name="defaultEnumTypeHandler" value="EnumOrdinalTypeHandler" /> 设置
通过自定义TypeHandler类实现#setParameter()和getResult()接口完成从javaType和jdbcType双向转换

MybatisExecutor执行器分类及区别？

SimpleExecutor：每次执行update或select都创建Statement对象，用完后立刻关闭
ReuseExecutor：执行update或select时以SQL作为key查找缓存的Statement对象，存在就使用，不存在就创建；用完后放入缓存Map<String, Statement>内
BatchExecutor：执行update操作调用addBatch方法将所有SQL都添加到批处理中，等待executeBatch方法统一执行。它缓存了多个Statement对象
CachingExecutor ：在三个执行器之上增加二级缓存的功能。
<setting name="defaultExecutorType" value="SIMPLE、REUSE、BATCH"> 分别使用上面三个执行器
<setting name="cacheEnabled" value=""> value=true时创建 CachingExecutor执行器

MyBatis如何执行批量操作？

用foreach标签：属性item，index，collection，open，separator，close。
用ExecutorType.BATCH

Mybatis的插件运行原理？如何自定义插件？

使用JDK动态代理插入代码。自定义ParameterHandler、ResultSetHandler、StatementHandler、Executor的插件
为需要拦截的接口生成代理对象以实现接口方法拦截功能，每当执行这4种接口对象的方法时，就会进入拦截方法，具体就是 InvocationHandler 的 #invoke(...)方法。
1.实现Interceptor接口实现intercept(...) 方法。
2.给插件添加注解指定要拦截哪接口的方法
3.在配置文件中配置插件

Mybatis如何分页？分页插件的原理？

不推荐：使用RowBounds对象对ResultSet结果集执行内存分页，而非数据库分页
推荐：在SQL输入分页参数；使用分页插件自动添加分页条件
原理：用插件接口实现自定义分页插件。拦截Executor的query方法，添加分页参数
常用分页插件：Mybatis-PageHelper MyBatis-Plus

Configuration:用于保存全局配置信息（如jdbc数据源、SQL 映射文件等），MyBatis 框架在启动完成以后会加载主配置文件，将配置信息转换为 Configuration 对象。
SqlSession：面向用户的API，提供了与数据库交互的方法，并封装了底层的 JDBC 操作。
Executor：SQL执行器,用于和数据库交互.SqlSession是面向用户的接口，实际的 SQL 执行是由 Executor 完成的。
MappedStatement:用于描述SQL配置信息，MyBatis 框架启动时，XML文件或者注解配置的SQL信息会被转换为MappedStatement对象(存储 SQL 映射信息（如 SQL 语句、参数映射、结果映射等）)注册到Configuration 组件中。
StatementHandler:封装了对JDBC中Statement 对象的操作，包括为Statement 参数占位符设置值，通过 Statement 对象执行 SQL语句
TypeHandler:类型处理器，用于Java类型与基础类型之间的转换。尤其是对于自定义类型、数据库的特殊类型（如日期、枚举等）需要特别处理时。
ParameterHandler:用于处理 SQL中的参数占位符，为参数占位符设置值。
ResultSetHandler:封装了对ResultSet对象的处理逻辑，将结果集转换为Java实体对象
