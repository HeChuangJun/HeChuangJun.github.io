Aug 5, 2018 | express

express框架是基于node.js的封装

hello world案例

//1.express的安装,在项目的目录下cmd执行npm i -S express
//2.引包
var express = require('express');
//3.创建服务器应用程序==http.createServcer
var app = express();
//4.当服务器收到get请求/的时候执行回调处理函数
app.get('/',function(req,res){
    res.send('hello express');
})
app.post('/',function(req,res){
    res.send('hello express');
})
//5.设置监听接口 server.listen
app.listen(3000, function(){
    console.log('app is running at port 3000');
})

封装自己的promise的ajax

function get(url,callback){
    return new Promise(function(resolve,reject){
        var oReq = new XMLHttpRequest()
        oReq.onload =fucntion(){
            callback && callback(oReq.responseText)
            resolve(oReq.responseText);

        }
        oReq.onerror =function(err){
            reject(err)
        }
        oReq.open("get",url,true)
        oReq.send()        
        })

    }
    //调用
    get('data.json',function(data){
        console.log(data)
    })
    //调用
    get('data.json')
        .then(function(data){
            console.log(data)
        })

express的中间件（处理请求的,本质是一个函数）

在express中，应用程序级别的中间件分类（同一个请求在不同的路由中的对象一致）

请求到来的时候会从第一个匹配的中间开始，进入中间件后，
    不用next是不会往下匹配中间件上的，与中间件的顺序无关
    任何请求路径和方法的中间件
    app.use(function(req,res,next){
        next();(进入下一个匹配的中间件)

    })
    指定开头的中间件/a开头的都进去的中间件
    app.use(‘/a’,function(req,res,next){
        next();(进入下一个匹配的中间件)

    })
    严格匹配请求方法get或者poes和请求路径/的中间件
    app.get('/'，function(req,res,next){

    })
    app.post('/'，function(req,res,next){

    })
