Aug 25, 2018 | webpack

webpack是常用的前端的前端自动化构建的工具

网站中引入的静态资源多了以后有什么问题？

网页加载速度慢，因为我们会发起许多静态资源的二次请求
    要处理错综复杂的依赖关系

如何解决上面的问题

合并、压缩、精灵兔、图片的base64编码
    requirejs和webpack解决复杂的依赖关系

如何完美实现上诉的两种解决方案

使用Culp（基于task任务的）、webpack（基于这个项目构建）
    借助webpack这个工具，可以完美实现资源的合并、打包、压缩混淆等诸多功能

什么是webpack

webpack是前端的一个项目构建工具，基于node.js开发出来的一个前端工具

开发webpack的项目的结构

webpack的安装

npm i webpack -g(安装node.js后再安装这个)
    在项目的根目录中运行npm i webpack --save-dev安装到项目依赖中

webpack的使用

推荐在main.js中引入所有需要的包，而其他页面只要引用webpack打包后的js即可

webpack的打包

webpack .\src\main.js .\dist\bundle.js(导出main的js)

webpack的功能

webpack能够处理js文件的相互依赖关系
    webpack能够处理js的兼容问题，把高级的、浏览器不能识别的语法转成浏览器能够正常识别的语法

webpack-dev-server实现热部署（依赖webpack）

安装：cnpm i webpack-dev-server -D 只能本地安装（不能直接用webpack当做脚本使用：非全局）
    使用：在package.json中声明script:{“dev”:"webpack-dev-server"}然后使用npm run dev
    webpack-dev-server打包生成的bundle.js,放在内存中，在项目根目录中找不到bundle.js，
    与根目录src等同级，所以我们引用budle.js使用根目录的bundle即可
    第一种方法：在package.json中声明

node和es6导出导入成员的形式

ES6中通过规范的形式，规定了ES6中如何导入和导出模块
    ES6中导入模块，使用import 模块名称 from '模块标识符' import '标识路径'
    ES6中使用export default 和export向外暴露成员
    export var  test1= ‘假假按揭’;可以导出多个，接受的时候不能改名，可以按需接受可以用as改名
    export default {‘msg’：123}//只能导出一个，接受能改名，
    import abc，{test1，test2} from '模块标识符'
    Node中使用var 名称 =require('模块标识符')
    module.export和export暴露成员
