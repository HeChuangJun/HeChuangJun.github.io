webpack

Aug 25, 2018 | webpack

网站中引入的静态资源多了以后有什么问题？

网页加载速度慢，因为我们会发起许多静态资源的二次请求
    要处理错综复杂的依赖关系

如何解决上面的问题

合并、压缩、精灵兔、图片的base64编码
    requirejs和webpack解决复杂的依赖关系

如何完美实现上诉的两种解决方案

使用Culp（基于task任务的）、webpack（基于这个项目构建）
    借助webpack这个前端自动化构建工具，可以完美实现资源的合并、打包、压缩混淆等诸多功能

什么是webpack

webpack是前端的一个项目构建工具，基于node.js开发出来的一个前端工具

开发webpack的项目的结构

webpack的安装

npm i webpack -g(安装node.js后再安装这个)
    在项目的根目录中运行npm i webpack --save-dev安装到项目依赖中

webpack的使用

推荐在main.js中引入所有需要的包，那么其他页面只要引用webpack打包之后的js即可

webpack的打包

webpack .\src\main.js .\dist\bundle.js(导出main的js)

webpack的功能

webpack能够处理js文件的相互依赖关系
    webpack能够处理js的兼容问题，把高级的、浏览器不能识别的语法转成浏览器能够正常识别的语法

webpack.config.js(放在项目的根目录下)

用于打包的时候不用指定webpack的打包出入口问题，只需简单输入webpack自动打包
    const path = require('path');
    module.exports ={
        entry:path.join(__dirname,'./src/main.js'),//webpack打包的路径
        output:{
            path:path.join(__dirname,'./dist'),//webpack打包后输出的目录
            filename:'bundle.js'//指定输出文件的名称
        }
    }

webpack-dev-server实现热部署（依赖webpack）

安装：cnpm i webpack-dev-server -D 只能本地安装（不能直接使用webpack当做脚本使用：不是全局的）
    使用：在package.json中声明script:{“dev”:"webpack-dev-server"}然后使用npm run dev
webpack-dev-server打包生成的bundle.js,不是放在实际的物理磁盘上，而是放在内存中，在项目根目录中找不到
bundle.js，他和根目录src等同级，所以我们引用budle.js使用根目录的bundle即可

webpack-dev-server常用命令参数（依赖webpack）

①在package.json中声明script:{“dev”:"webpack-dev-server"}
“dev”:"webpack-dev-server --open --port 3000 --contentBase src --hot"
--hot:代码修改浏览器自动刷新，且不会重新生成bundle.js，只是打个补丁就算了
②在webpack.config.js中多加一项
const webpack = require('webpack');
devServer：{open：true，port：3000，contentBase：‘src’，hot：true}
plugins:[
    new webpack.HotModuleRepalcementPlugin()//新建一个热更新的模块对象
]

webpack将html加入到内存中（插件，只要是插件都要在plugins中声明）

两个作用，在内存中生成内存页面和把打包好的bundle。js追加到页面中去
安装 cnpm i html-webpack-plugins -D
    然后在在webpack.config.js中多加一项
    const htmlwebpackplugins = require('html-webpack-plugins')
    plugins:[
        new htmlwebpackplugins({//创建一个在htmk页面插件
            template：path.join(__dirname,'./src/index.html')//将制定页面生成到内存中
            filename：‘index.html’

        })
    ]

处理非js文件

安装cnpm i style-loader css-loader less-loader sass-loader -D
cnpm i less -D     cnpm i url-loader file-loader -D    
然后在在webpack.config.js中多加一项
    module：{//配置所有第三方模块加载器
        rules：[//所有第三方模块的匹配规则
            {test:/\.css$/,use:['style-loader','css-loader']}//配置匹配的规则
            //从右向左调用
            {test:/\.less$/,use:['style-loader','css-loader','less-loader']}//配置匹配的规则
            {test:/\.scss$/,use:['style-loader','css-loader','sass-loader']}//配置匹配的规则
            {test:/\.(jpg|png|gif|bmp|jpeg)$/,use:'url-loader?limit=xxx&name=[hash:8]-[name].[ext]']}//配置匹配的规则
            url(./images/hahah.jpg)
            //limit单位是字节，大于等于给定值不会转成base64，反之会转
            name属性表示引用的图片名字，默认使用hash值，[name].[ext]用原来的名字和扩展名
            名字如果一样会覆盖，以最后一次声明为准，hash表示使用hash值拼到文件名中
            {test:/\.(ttf|eot|svg|woff|woff2)$/,use:['url-loader']}//处理字体文件
        ]
    }

引入样式

在node_module中使用包的时候不需要写node_module
    package.json文件不能写注释，注意全局命令的使用必须得有包，node报错的时候重装依赖就行

ES6及以上的新语法的处理

语法太新webpack处理不来，需要借助第三方loader来转换，然后把结果交给webpack打包到
    bundle.js中
    在webpack中，安装Babel的两套包
    cnpm i label-core babel-loader babel-plugin-transform-runtime -D
    cnpm i label-preset-env babel-preset-stage-0 -D
    然后在在webpack.config.js中多加一项
    plugins:[
        {test:/\.js$/,use:['babel-loader']，exclude：/node_modules/}
        //排除node_modules较少cpu消耗，不排除导致node_modules不能用
    ]
    在项目根目录中新建一个叫做.babelrc的Babel配置文件
    {
        "presets":["env","stage-0"]
        "plugins":["transform-runtime"]
    }

webpack与Vue连用

①import Vue from 'vue'//只是导入了runtime-only的方式，没有提供像网页那种使用方法（一般使用这个）
    然后在在webpack.config.js中多加一项
    resolve:{
        alias:{
            "vue$":"vue/dist/vue.js"//设计被导入包的路径
        }

    }
②import Vue from '../node_modules/vue'    //直接引入最全的vue包，能像原来页面开发那样开发（不推荐）

node和es6导出导入成员的形式

ES6中通过规范的形式，规定了ES6中如何导入和导出模块
    ES6中导入模块，使用import 模块名称 from '模块标识符' import '标识路径'
    ES6中使用export default 和export向外暴露成员
    export var  test1= ‘假假按揭’;可以导出多个，接受的时候不能改名，可以按需接受可以用as改名
    export default {‘msg’：123}//只能导出一个，接受能改名，
    import abc，{test1，test2} from '模块标识符'
    Node中使用var 名称 =require('模块标识符')
    module.export和export暴露成员
