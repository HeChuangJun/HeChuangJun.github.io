vue.js介绍

vue.js和angular.js和react.js并称前端三大框架
vue.js手机一套构建用户界面的框架，只关注视图层，易上手，便于与第三方库或既有的项目整合
vue.js有配套的第三方库，可以整合起来做大型项目的开发
前端主要负责mvc中的v的这一层l；主要与页面打交道

vue.js的优势

vue.js让用户不再或者减少操作dom元素,提高渲染效率，双向绑定数据，更关注业务逻辑

框架与库的区别

框架:是一套完整的解决方案，如node中的express，更换框架难
    库（插件）:提供某个小功能,切换容易

mvc与mvvm的区别

mvc是后端的分层开发概念
mvvn是前端视图层的盖帘，主要关注与视图层分离，mvvn把前端的视图层分成了model、view，vm viewmodel

自定义全局指令

Vue.directive('指令的名称（不需v-前缀，调用需要v-）'，对象包括指令的函数（钩子函数）)
// 注册一个全局自定义指令 v-focus
    Vue.directive('focus', {
        // 当绑定元素插入到 DOM 中执行（1次）dom操作和样式都有用
        inserted: function (el) {// 聚焦元素el.focus()}
        bind:funciton(el){}//每当指令绑定到元素上的时候，就执行（1次）
        //在元素刚绑定，还没有插入到dom中去，调用focus没有作用
        //因为元素插入dom之后才能获取焦点，dom操作没有用，样式有用
        update:function(el){}//当vnode更新的时候就会执行（可能多次执行）
        componentUpdate(){}
        unbind:function(){}
    })
    //钩子函数包括的参数有
    el: 指令所绑定的元素，可以用来直接操作 DOM 。
    binding: 一个对象，包含以下属性：
        name: 指令名，不包括 v- 前缀。
        value: 指令的绑定值， 例如： v-my-directive="1 + 1", value 的值是 2。（表达式的值）
        oldValue: 指令绑定的前一个值，仅在 update 和 componentUpdated 钩子中可用。无论值是否改变都可用。
        expression: 绑定值的表达式或变量名。 例如 v-my-directive="1 + 1" ， expression 的值是 "1 + 1"。
        （原样输出，包括字符串的''或者""）
        arg: 传给指令的参数。例如 v-my-directive:foo， arg 的值是 "foo"。
        modifiers: 一个包含修饰符的对象。 例如： v-my-directive.foo.bar, 修饰符对象 modifiers 的值是 { foo: true, bar: true }。
    vnode: Vue 编译生成的虚拟节点。
    oldVnode: 上一个虚拟节点，仅在 update 和 componentUpdated 钩子中可用。

Vue的生命周期

在Vue实例创建、运行、到销毁期间，总是伴随着各种各样的事件，统称为生命周期

生命周期函数使用

new Vue({
    data: {
    a: 1
    },
    beforeCreate：function(){
        //data和methods都还没有被初始化
    }
    created: function () {//一般在此位置获取数据库的数据
        //使用this.操作data和methods
    }
    beforeMount：function(){
        //模板已经在内存中编辑完成了，尚未把模板渲染到页面
        //页面中的元素还没有被真正替换过来，只是写的一些字符串模板{{xxx}}
    }
    Mounted：function(){
        //内存中的模板已经真实地挂载到页面中并显示了，vue实例已经被创建好了
        //如果要操作dom节点，最早要在这函数中执行
        //此时组件已经脱离了创建阶段，进入运行阶段
        //如果没有其他操作的话，vue实例就会一直留在内存中
    }
    //运行中的两个事件
    beforeUpdate(){//数据一旦更新且视图改变前触发，不管视图是不是改变
        //data是最新的，但是视图不是最新的
    }
    updated(){
        //数据和视图上的元素都是最新的，视图和数据已经同步
    }
    beforeDestory(){
        //vue实例从运行阶段进入到销毁阶段，此时所有的data和method等都处于可用状态
        //还没有执行到销毁
    }
    destoryed(){
        //vue实例已经销毁，所有的组件不可用
    }

    })

jsonp的原理

jsonp的产生原因

由于浏览器的安全性限制，不允许ajax访问协议不同、域名不同、端口号不同的数据接口
    浏览器认为这种访问不安全，可以通过script标签的形式将src地址指向接口地址（scrpit不存咋跨域限制）
    这种方式称为jsonp（只支持get请求）

jsonp使用的时候相当于访问请求获得js脚本，所以服务端需要返回相应的脚本字符串

一般采用jsonp请求加入请求参数callback=回调方法名，然后后端解析返回callback（后端结果）来调用前端的回调方法完成jsonp请求

jsonp前端

<script src="http://127.0.0.1:8091/test/test?callback=show"></script>
<script >
    function show(data){
        console.log(data);
    }
</script>

java后端

@RequestMapping("/test")
    @ResponseBody
    public String fun(HttpServletRequest request,HttpServletResponse response) {
    User u=new User("junye", "1");
    String parameter = request.getParameter("callback");
    JSONObject user = JSONObject.fromObject(u);    
    return parameter+"("+user.toString()+")";
    }

Vue的组件

组件的出现就是为了拆分vue实例的代码量的，
    能够让我们以不同的组件划分不同的
    模块，需要的功能调用相应的组件即可

模块化和组件化的区别

模块化：从代码逻辑的角度进行划分（node.js）方便代码的分层开发，保证每个功能模块的职能单一
    组件化：从ui的角度划分(vue.js)前端的组件化方便ui组件的重用

Vue的computed计算值

<script type="text/javascript" src="./vue.js"></script>
<div id="app">
    <input type="text" v-model="firstName">
    <input type="text" v-model="MiddleName">
    <input type="text" v-model="lastName">
    <input type="text" v-model="fullName">//调用计算属性

</div>

<script type="text/javascript">vm = new Vue({
el:'#app',
data:{
    firstName:'',
    lastName:'',
    MiddleName:''
},
computed:{
    fullName:function(){//只要这个函数中的计算属性发生变化后只要修改后就会触发
        //计算属性会被缓存起来，第二次引用的时候就会读取缓存而不是调用这个
        return this.firstName+this.lastName+this.MiddleName;
    }
}
});
</script>

Vue的watch、methods和computed之间的对比

computed属性的结果会被缓存，除非依赖的响应属性发生变化才会计算，主要当做属性来使用
    methods方法表示一个具体的操作，主要书写业务逻辑
    watch一个对象，键是需要观察的表达式，值是对应函数的回调函数，主要用来监听某些特定数据的变化
    从而进行某些具体的业务逻辑操作，可以看做是computed和methods的结合体
