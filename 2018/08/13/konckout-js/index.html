konckout.js

Aug 13, 2018 | konckout.js

konckout是一个前端的js库，已经实现mvvm的模式，相对于vue.js框架比较好管理
虽然是后端工程师，但因工作需要就学学konckout.js吧

konckout.js的介绍

什么是mvvm详见vue.js前端框架的介绍

konckout.js的hello world案例

The name is <span data-bind="text: personName"></span>//view层
    var vm = {//myViewModel是view model层
        personName: 'Bob',//model层
        personAge: 123//属性存在覆盖，以最后一次声明为准
    };
    ko.applyBindings(vm);

konckout.js的hello world案例2(重要)

根据js的特性，helloworld案例还可以这样写
    The name is <span data-bind="text: personName"></span>//view层
    var vm = function{//myViewModel是view model层
        personName: 'Bob',//model层
        personAge: 123//属性存在覆盖，以最后一次声明为准
    };

所有api详解（视图修改会导致vm中的m修改！！！）

ko.applyBindings(?,[?]);

第一个参数传入vm对象（必须），第二个参数传入document元素（可选，限制使用范围）
    例如ko.applyBindings(vm,document.getElementById('someElementId'));

“单向绑定”(使用applyBindings绑定后修改m不会导致v变化)

The name is <span data-bind="text: personName"></span>
    var vm = function(){
        var self=this;
        self.personName='Bob',//这样声明的话，修改该值不会导致视图改变
        self.personAge=123//属性存在覆盖，以最后一次声明为准
    };
    ko.applyBindings(new vm());
    window.onload = function(){
        vm.personName='junye';//视图不会有任何变化
    }

“双向绑定”(使用applyBindings绑定后修改m导致v变化)

The name is <span data-bind="text: personName"></span>//view层
    var vm = function(){//myViewModel是view 
        var self=this;
        self.personName=ko.observable('hello world!'),//这样声明的话，修改该值会导致视图改变
        self.personAge=123//属性存在覆盖，以最后一次声明为准
    };
    ko.applyBindings(new vm());
    window.onload = function(){
        vm.personName('junye');//修改双向绑定值，视图变化！
        vm.personName='junye';//这样修改视图不会变化！
        vm.personName();//获取双向绑定的值
    }

强制刷新（只要m中的值被修改）全部支持

vm.personName.extend({ notify: 'always' });

延时加载（当刷新比较频繁的时候）全部支持

vm.personName.extend({ rateLimit: 50 });//50毫秒后刷新

双向绑定数组（支持延时加载）

（对于数组来说实现双向绑定必须vm.数组名.数组方法 来修改数组）
（而不是vm.数组名().数组方法 修改数组，虽然两种方法都能修改数组，
但是只有第一种方法满足我们的要求，因为第一种是js原生方法）
The name is <span data-bind="text: personName"></span>//view层
    var vm = function(){//myViewModel是view model层
        var self=this;
        self.personName=ko.observableArray(['aaaa','bbbb']),//这样声明的话，修改该值会导致视图改变
        self.personAge:=123//属性存在覆盖，以最后一次声明为准
    };
    ko.applyBindings(new vm());
    window.onload = function(){
        vm.personName.push('xxxxx');//修改双向绑定值，视图变化！
        vm.personName().push('xxxxx')//这样修改视图不会变化！
        vm.personName();//获取双向绑定的数组本身
        vm.personName().length;//获取数组长度
        vm.personName()[];//获取数组的第一个
        vm.personName().indexOf('aaaa');返回对象在数组中的位置，没有则-1
        vm.personName().slice(1,3);//删除并返回数组指定开始和结束位置的函数
        vm.personName.slice(1,3);//两种写法一致
        vm.personName().push('avdc');//添加元素到数组末尾
        vm.personName().pop();//删除并返回最后一个数组的对象
        vm.personName().unshift('dada');//添加一个元素到数组头部
        vm.personName().shift();//删除第一个元素并返回
        vm.personName().reverse();//反转数组的顺序并且返回observableArray 
        vm.personName().sort();//返回有序数组observableArray
        vm.personName().sort(function(){});//自定义有序数组的转换
        vm.personName().replace(oldItem，newItem);//替换数组中第一个匹配的旧元素
        vm.personName().remove(Item);//删除所有匹配的数组并返回一个数组
        vm.personName().remove(function (item) { return item.age < 18; } );
        //删除所有小于18的元素并返回新数组
        vm.personName().removeAll(['Chad', 132, undefined])
        //删除所有匹配数组内容的元素并返回新的数组
        vm.personName().removeAll();//删除所有数组并返回新数组
    }

计算方法ko.computed(function(){})（一个data-bind监控其他两个data-bind的函数）

只要其中一个data-bind发生变化，计算方法中的data-bind所在的view就会显示
    The name is <span data-bind="text: personName"></span>//view层
    The personAge is <span data-bind="text: personAge"></span>//view层
    The person is <span data-bind="text: personName"></span>//view层
    var vm = funtion(){//myViewModel是view model层
        this.personName=ko.observableArray(['aaaa','bbbb']),//这样声明的话，修改该值会导致视图改变
        this.personAge=123//属性存在覆盖，以最后一次声明为准
        this.person=ko.computed(function() {
            return this.firstName() + " " + this.lastName();
        }, this);
    };
    ko.applyBindings(new vm());

ko.purecomputed(function(){})，

这是一个更加高效的方法，用于替代不涉及修改其他对象的computed方法
    防止内存泄漏和减少计算开销来提高性能

ko.computed能通过判断来决定那个组件显示

ko.isObservable()- returns true for observables, observable arrays, and all computed observables.
    ko.isWritableObservable() - returns true for observables, observable arrays, 
    and writable computed observables (also aliased as ko.isWriteableObservable).

data-bind=”click: incrementClickCounter”

或者data-bind="click: xxx.xxx"
    还可以绑定特定的function
    myFunction: function(data, event) {
        if (event.shiftKey) {
            //do something different when user has shift key down
        } else {
            //do normal action
        }
    }
    或者传入多个参数
    <button data-bind="click: function(data, event)
    { myFunction('param1', 'param2', data, event) }">

    等价于以下的情况
    <button data-bind="click: myFunction.bind($data, 'param1', 'param2')">
        Click me
    </button>

valueAllowUnset的案例

<p>

valueAllowUnset: true"></select>
</p>

<script type="text/javascript">
var viewModel = {
countries: ['Japan', 'Bolivia', 'New Zealand'],

</script>
在上面的例子中selectedCountry会保留Latvia，然后下联显示空白
因为没有匹配的选项
如果valueAllowUnset是false，selectedCountry会被写成undefined
以便匹配那个Choose one...

value的特殊绑定

<!-- Two-way binding. Populates textbox; syncs both ways. -->
    <p>First value: <input data-bind="value: firstValue" /></p>
    //改变视图和model
    <!-- One-way binding. Populates textbox; syncs only from textbox to model. -->
    <p>Second value: <input data-bind="value: secondValue" /></p>
    //改变model，但是不可见
    <!-- No binding. Populates textbox, but doesn't react to any changes. -->
    <p>Third value: <input data-bind="value: secondValue.length > 8" /></p>
    //什么都不改变
    <script type="text/javascript">
    var viewModel = {
    firstValue: ko.observable("hello"), // Observable
    secondValue: "hello, again"         // Not observable
    };
    </script>
