多线程

多线程

进程与线程

进程是处于运行过程中的程序，并且具有一定独立功能
    线程：线程是进程中的一个执行单元，负责当前进程中程序的执行
    一个程序运行后至少有一个进程，一个进程中包含至少一个线程
    单线程程序：若有多个任务只能依次执行
    多线程程序：若有多个任务可以同时执行
    主线程：jvm启动后，必然有一个从main方法开始的，一直执行到main方法结束的线程

程序运行原理

分时调度：所有线程轮流使用 CPU 的使用权，平均分配每个线程占用 CPU 的时间。
    抢占式调度(Java)：优先让优先级高的线程使用 CPU，线程优先级相同，随机选择一个线程执行
    CPU(中央处理器)使用抢占式调度模式在多个线程间进行着高速的切换
    对于CPU的一个核而言，某个时刻，只能执行一个线程，
    而 CPU的在多个线程间切换速度相对我们的感觉要快，看上去就是在同一时刻运行
    多线程程序并不能提高程序的运行速度，但能够提高程序运行效率，让CPU的使用率更高

Thread类是程序中的执行线程，实现了Runnable接口

Java 虚拟机允许应用程序并发地运行多个执行线程。
    Thread()分配新的Thread对象
    Thread(String name)分配新的Thread对象,将指定的name作为线程名称
    Thread(Runnable target)分配新的Thread对象，以便将target做为其运行对象
    Thread(Runnable target,String name)~指定的name作为线程名称

    void start()使该线程开始执行，Java虚拟机调用该线程的run方法
    void run()该线程要执行的操作
    (Thread.)static void sleep(long millis)在指定的毫秒数内让当前正在执行的线程休眠
    static Thread currentThread()返回当前正在执行的线程对象的引用
    String getName()返回该线程的名称，
    主线程的名称:main;自定义的线程:Thread-0,线程多个时，数字顺延。如Thread-1......

创建线程的方式

继承Thread类

线程对象调用run方法和调用start方法区别？
调用run方法不开启线程(对象调用方法),调用start方法开启线程,让run方法在线程中执行
我们为什么要继承Thread类，并调用其的start方法才能开启线程呢？
继承Thread类：因为Thread类用来描述线程，具备线程应该有功能
创建线程的目的是什么？
是为了让多部分代码实现同时执行,线程创建并执行需要给定线程要执行的任务
    主线程，它的任务定义在main函数中。自定义线程需要执行的任务都定义在run方法中
为什么不直接创建Thread类的对象呢？
因为即使创建Thread对象并调用start方法，run方法也没有我们需要线程执行的任务
示例
public class Demo01 {
    public static void main(String[] args) {
    MyThread mt = new MyThread("新的线程！");
    mt.start();
    for (int i = 0; i < 10; i++) {
        System.out.println("main线程！"+i);
        }
        }
    }

public class MyThread extends Thread {
    public MyThread(String name) {
        super(name);
    }
    @Override
    public void run() {
        for (int i = 0; i < 10; i++) {
            System.out.println(getName()+"：正在执行！"+i);
            }
        }
    }

实现Runnable接口

避免了继承Thread类的单继承局限性
    实现Runnable接口的方式，更加的符合面向对象，线程分为线程对象和线程任务。
    继承Thread类，线程对象和线程任务耦合。Thread类的子类对象，既是线程对象，又有线程任务
    实现runnable接口，将线程任务单独分离出来封装成对象，类型就是Runnable接口类型。
    Runnable接口对线程对象和线程任务进行解耦。

示例

public class Demo02 {
    public static void main(String[] args) {
    //创建线程执行目标类对象
    Runnable runn = new MyRunnable();
    //将Runnable接口的子类对象作为参数传递给Thread类的构造函数
    Thread thread = new Thread(runn);
    Thread thread2 = new Thread(runn);
    //开启线程
    thread.start();
    thread2.start();
    for (int i = 0; i < 10; i++) {
    System.out.println("main线程：正在执行！"+i);
    }
    }
    }    
    public class MyRunnable implements Runnable{
    @Override
    public void run() {
    for (int i = 0; i < 10; i++) {
    System.out.println("我的线程：正在执行！"+i);
    }
    }
    }
当执行线程的任务结束了，线程自动在栈内存中释放了。当所有执行线程都结束了，进程就结束了

线程匿名内部类的使用

创建线程对象时，重写Thread类中的run方法

new Thread() {
    public void run() {
    for (int x = 0; x < 40; x++) {
    System.out.println(Thread.currentThread().getName()+ "...X...." + x);
    }
    }
    }.start();

实现Runnable接口，重写Runnable接口中的run方法

Runnable r = new Runnable() {
    public void run() {
    for (int x = 0; x < 40; x++) {
    System.out.println(Thread.currentThread().getName()+ "...Y...." + x);
    }
    }
    };
    new Thread(r).start();

线程池

容纳多个线程的容器(使线程可以反复使用),无需反复创建线程而消耗过多资源
    ①线程生命周期开销问题(创建和销毁线程花费的时间和系统资源都相当大)
    ②资源不足问题（线程太多过度消耗内存或‘切换过度’）
    适当的调整线程中的线程数目可以防止出现资源不足的情况。
    ③运行速度问题（响应快）,通过对多个任务重复使用线程,
    线程创建的开销就被分摊到了多个任务上了，
    而且由于在请求到达时线程已经存在，所以消除了线程创建所带来的延迟。

线程池的使用

①Executors：线程池创建工厂类
    public static ExecutorService newFixedThreadPool(int nThreads)：返回线程池对象
    ExecutorService service = Executors.newFixedThreadPool(2);//包含2个线程对象
    ②实现Runnable接口并覆盖void run()方法，创建该对象（非多态）
    ②实现Callable<T>接口覆盖Object call()方法，创建该对象（非多态）
    ③调用线程池类的方法
    Future<?> submit(Runnable task)获取线程池中某个线程对象并执行线程中run()
    <T> Future<T> submit(Callable<T> task)获取线程池中某个线程对象并执行线程中call()
    ④Future接口：用来记录线程任务执行完毕后产生的结果。
    V get() 获取Future对象中封装的数据结果
    对于实现了callable接口的线程类，线程执行成功则返回覆盖call方法执行的返回的object
    对于实现了Runnable接口的线程类，线程执行成功则返回覆盖run方法执行的返回的null
    关闭线程池ExecutorService  void shutdown();线程自动关闭

线程安全

多线程每次运行的结果与单线程运行的结果是一样的，且其他变量的值也是预期的一样的
    线程安全问题都是由全局变量及静态变量引起的。
    若每个线程中对全局变量、静态变量只有读操作，而无写操作，这个全局变量是线程安全的；
    若有多个线程同时执行写操作，一般都需要考虑线程同步，否则的话就可能影响线程安全

线程同步:解决线程安全问题

同步代码块synchronized (锁对象) {同步代码}
    同步代码块中的锁对象可以是任意的对象；多个线程时，同一个锁对象才能够保证线程安全。
    同步方法public synchronized void 方法名(){}在run中调用这个方法，锁对象是 this
    静态同步方法public static synchronized void method(){}，锁对象是 类名.class

死锁：同步锁使用的弊端

当线程任务中出现了多个同步(多个锁)时，如果同步中嵌套了其他的同步。程序会出现无限等待
    是由于进入同步代码块的线程不释放锁而引起的，如AB锁交叉
        synchronzied(A锁){
            synchronized(B锁){         
        }
        }

死锁解决办法（使用lock接口）

Lock ck = new ReentrantLock();//ReentrantLock实现了Lock接口多态
    ck.lock();获取锁
    同步代码块
    ck.unlock();释放锁

线程同步执行过程

①线程进同步前判断锁，没锁进不去，有锁进去并获取锁，出同步释放锁，导致线程运行速度下降
    ②没有锁的线程，不能进入同步，③在同步中的线程，不出去同步，不会释放锁，即使休眠了

等待唤醒机制（同步中才有效）

等待唤醒机制：使各个线程能有效的利用资源。
    Object中的相关方法
    wait（） :等待，将正在执行的线程释放其执行资格和执行权，并存储到线程池中
    notify（）：唤醒，唤醒线程池中被wait（）的线程，一次唤醒一个，而且是任意的
    notifyAll（）： 唤醒全部：可以将线程池中的所有wait() 线程都唤醒

sleep()和wait()方法的区别

sleep: 不释放锁对象, 释放CPU使用权、在休眠的时间内，不能唤醒
    wait(): 释放锁对象, 释放CPU使用权、在等待的时间内，能唤醒
