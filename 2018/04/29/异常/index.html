异常

异常：Java代码在运行时期发生的问题

Java把异常信息封装成一个类。异常时会创建异常类对象并抛出异常信息(如异常位置、原因)

异常的继承体系

Throwable所有错误或异常的超类

子类Exception编译期异常：编译程序时出现的问题
程序中可能会产生的异常,并要求对产生的异常进行合理的异常处理,若不处理,程序结束运行
    ①子类RuntimeException及其子类异常,都属于运行时期异常。只在程序运行过程中出现
    ②方法中抛出运行时期异常,方法定义中无需throws声明,调用者也无需处理此异常
    运行时期异常一旦发生,需要程序人员修改源代码.
    常见异常
    ArrayIndexOutOfBoundsException()不带详细信息的ArrayIndexOutOfBoundsException
    ArrayIndexOutOfBoundsException(int index)具有指示非法索引的新的~类
    ArrayIndexOutOfBoundsException(String s)具有指定详细信息的~类
    NullPointerException()不带详细信息的NullPointerException
    NullPointerException(String s)带指定详细信息的NullPointerException
子类Error：表示java程序中可能会产生的严重错误。
通常没有具体的处理方式，程序将会结束运行。Error错误的发生往往都是系统级别的问题，
    都是jvm所在系统发生的，并反馈给jvm的。无法针对处理，只能修正代码

异常处理：当产生异常时，必须有处理方式。要么捕获，要么声明。

抛出异常throw：在方法函数头抛出异常对象，交给调用者处理

throw new 异常类名(参数);

声明异常throws：在方法函数内部抛出异常对象，交给调用者去处理

修饰符 返回值类型 方法名(参数) throws 异常类名1,异常类名2… {}

throw抛出异常，交给调用者处理，本身不捕获就要throws抛出

catch捕获异常

Java中对异常有针对性的语句进行捕获，可以对出现的异常进行指定方式的处理
    try {
        //需要被检测的语句。
    }
    catch(异常类 变量) { //参数。
    //异常的处理语句。处理异常
    }
    finally {
    //一定会被执行的语句。释放资源
    }
    异常处理组合方式
    try catch finally组合:检测异常，并传递给catch处理，并在finally中进行资源释放
    try catch组合:对代码进行异常检测，并对检测的异常传递给catch处理.对异常进行捕获处理
    try 多个catch组合:对代码进行异常检测，并对检测的异常传递给catch处理
    多个catch中的异常不能相同，子类异常要求在上面的catch处理,父类异常在下面的catch处理
    try finally 组合:异常是没有捕获处理的，finally，只为关闭资源
    异常的处理，指处理异常的一种可能性，即有了异常处理的代码，不一定会产生异常
    如果没有产生异常,则代码正常执行,如果产生异常,则中断当前执行代码,执行异常处理代码

异常在方法重写（覆盖）的细节

如果父类方法声明异常，子类只能声明父类异常或者该异常的子类，或者不声明
    当父类方法声明多个异常时，子类覆盖时只能声明多个异常的子集
    当被覆盖的方法没有异常声明时，子类覆盖时无法声明异常的。
问题：接口中没有声明异常，而实现类覆盖方法时发生了异常，怎么办？
无法进行throws声明，只能catch的捕获。万一问题处理不了呢？
    catch中继续throw抛出，但是只能将异常转换成RuntimeException子类抛出。

异常方法

String getMessage()返回此throwable的详细消息字符串
    void printStackTrace()将此throwable及其追踪输出至标准错误流
    String toString()返回此throwable的简短描述

自定义异常

Class 异常名 extends Exception{ 
    //编译时异常继承Exception，运行时异常继承RuntimeException。
    public 异常名(){
    super();
    }
    public 异常名(String s){ 
    super(s); 
    }
    }
    继承Exception,必须throws声明,一声明就告知调用者捕获,问题处理了调用者的程序会继续执行
    继承RuntimeExcpetion,不需throws声明的，这时调用是不需要编写捕获代码的，
    因为调用根本就不知道有问题。一旦发生NoAgeException，调用者程序会停掉，
    并由jvm将信息显示到屏幕，让调用者看到问题，修正代码。
