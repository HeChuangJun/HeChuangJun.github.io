面向对象的三大特征

面向对象的三大特征

封装:隐藏实现细节，对外提供可以访问的方式。便于调用者的使用

表现:方法（最基本封装体，封装了多行代码）、类（封装了多个方法）、private修饰的变量
    好处:提高了代码的复用性、提高了安全性
    this关键字
        在成员变量名前面加上this.来区别成员变量和局部变量
        this代表的是本类创建的对象，具体代表哪个对象呢？
        哪个对象调用了this所在的方法，this就代表哪个对象，在对象创建时就存在
    private关键字
        类中不需要对外提供的属性和方法都私有化,并提供setXxx,getXxx方法对其进行访问
    注意：私有仅仅是封装的体现形式而已

继承：在现有类基础(父类)上构建新类(子类)，子类自动拥有父类所有非私有属性和方法

格式：class 子类 extends 父类 {}
    好处：提高了代码的复用性，提高软件开发效率，类与类之间产生关系，是多态的前提
    弊端：类与类之间的耦合度过高
    一个类不可以继承多个类（单继承），一个类只能有一个直接父类
    一个类可以被许多类继承
    多层继承是可以的，即一个类的父类可以再去继承另外的父类
    子类和父类是一种相对概念
    所有的类都直接或者间接的继承了Object类
    子父类中成员变量的特点
        默认访问子类的[同名]成员变量，不存在则访问父类非私有[同名]成员变量
        子类访问父类中的同名成员变量：super.父类中的成员变量
    子父类中成员方法特点
        默认执行子类的方法，不存在则执行父类中非私有的相应的方法
    override重写、复写、覆盖
        子类与父类方法同名时，直接调用子类的方法不会理父类的方法
        子类相同方法需要特定功能就重写父类中非私有的方法
        需要父类功能时加上super.父类中的同名方法()获得父类功能
    super:用来表示当前对象中包含的父类对象空间的引用
        子类方法覆盖父类方法，必须要保证权限大于等于父类权限

多态：子类对象(实现类对象)既可以给这个子类(实现类对象)引用变量赋值，又可以给这个子类(实现类对象)的父类(接口)变量赋值，最终多态体现为父类引用变量可以指向子类对象

多态前提：①子父类关系或者类实现接口关系②方法重写③父类引用指向子类对象

定义
普通类多态：父类 变量名 = new 子类();
    抽象类多态：抽象类 变量名 = new 抽象类子类();
    接口多态：接口 变量名 = new 接口实现类();
多态成员方法和变量的特点：方法的运行看右边，其他（编译、运行）都看左边
instanceof关键字：判断某个对象是否属于某种数据类型
boolean  b  = 对象(空格)instanceof(空格)数据类型;

转型：降低程序的耦合性

向上转型：当有子类对象赋值给一个父类引用（多态本身）
父类类型  变量名 = new 子类类型();
    好处：隐藏了子类类型，提高了代码的扩展性
    弊端：只能使用父类共性的内容，而无法使用子类特有功能
向下转型：一个已经向上转型的子类对象可以使用强制类型转换的格式
将父类引用转为子类引用，如果是直接创建父类对象，是无法向下转型的！
    子类类型 变量名 = (子类类型) 父类类型的变量;
    好处：可以使用子类特有功能
    弊端：需要面对具体的子类对象；容易发生ClassCastException类型转换异常。
    在转换之前必须做类型判断
选用
当不需要面对子类类型时，通过提高扩展性，或者使用父类的功能就能完成相应的操作
    当要使用子类特有功能时，就需要使用向下转型

抽象类与接口

抽象类：被abstract关键字修饰的类是抽象类，定义了抽象方法的类

定义

抽象类：abstract class 类名 {}
    抽象方法：public abstract 返回值类型 方法名(参数);

注意事项

抽象类可以不定义抽象方法。但抽象方法一定要定义在抽象类中
    抽象类不可以直接创建对象，原因：调用抽象方法没有意义。
    只有覆盖了抽象类中所有抽象方法后，其子类才可以创建对象。否则该子类还是一个抽象类
    抽象类都是父类，因为不断抽取而来的。
    抽象类的存在的意义是为了不让该类创建对象,方法可以直接让子类去使用
    abstract不可以和private、final、static共存

子类①实现了抽象方法的具体类②抽象类

接口：功能的集合，是比抽象类更为抽象的”类”

接口只描述所应该具备的方法，并没有具体实现，具体的实现由接口的实现类来完成。
    将功能的声明与实现分离，优化了程序设计
    定义接口所在的仍为.java文件，声明时使用interface关键字的编译后仍然会产生.class文件。
    可以将接口看做是一种只包含了功能声明的特殊类

定义

public interface 接口名 {
        public static final A_B=值;//常量，接口变量，不可被修改
        public abstract 返回值类型 方法名(数据类型 变量名，数据类型 变量名);//抽象方法
    }

特点&好处

接口不可以创建对象
    通过多态的方式，由子类来创建对象，接口多态    
    子类必须覆盖掉接口中所有的抽象方法后，子类才可以实例化。否则子类是一个抽象类

好处

接口的出现扩展了功能。
    接口其实就是暴漏出来的规则。
    接口的出现降低了耦合性，即设备与设备之间实现了解耦。

接口与类、接口与接口的关系

类实现接口（接口与类）多实现
class 类 implements 接口1,接口2 {重写接口中方法} 
    解决了多继承的弊端呢？
    弊端：多继承时，当多个父类中有相同功能时，子类调用会产生不确定性。
    核心原因就是在于多继承父类中功能有主体，而导致调用运行时，不确定运行哪个主体内容
    为什么多实现能解决了呢？因为接口中的功能都没有方法体，由子类来明确
接口的多继承（接口与接口）
interface Zi extends Fu1,Fu2,Fu3{
        void show3();
    }
多个接口之间可以使用extends进行继承
由于接口中的方法是抽象方法，子类实现后也不会发生调用的不确定性
类继承类同时实现接口
接口避免了单继承的局限性。父类中定义的事物的基本功能。
接口中定义的事物的扩展功能

接口和抽象的异同

相同点:
都位于继承的顶端,用于被其他类实现或继承;
    都包含抽象方法,其子类都必须覆写这些抽象方法;
    都不能直接实例化对象;直到抽象方法都覆盖完了才可以创建对象
区别:
①一个类只能继承一个直接父类(可能是抽象类),却可以实现多个接口;
    (接口弥补了单继承)
    ②抽象类为部分方法提供实现,避免子类重复实现这些方法,提高代码重用性;
    接口只能包含抽象方法;
    抽象类是这个事物中应该具备的你内容, 继承体系是一种 is..a关系
    接口是这个事物中的额外内容,继承体系是一种 like..a关系

二者的选用:

优先选用接口,尽量少用抽象类;
    需要定义子类的行为,又要为子类提供共性功能时才选用抽象类;

①Java为我们提供好的类API

②自定义类：java代码映射成现实事物的过程就是定义类的过程

类的定义：创建java文件，与类名相同

public class 类名{
    访问修饰符    数据类型  属性名称1；
    访问修饰符    数据类型  属性名称2；
        方法;
    }

方法

修饰符 返回值类型(void) 方法名([参数类型 参数名1,参数类型 参数名2,．．]//参数列表){ 
    执行语句
            ……… 
        [return 返回值;]
    }
    修饰符：有对访问权限进行限定的，有静态修饰符static，还有最终修饰符final等
    返回值类型：用于限定方法返回值的数据类型
    参数类型：用于限定调用方法时传入参数的数据类型
    参数名：是一个变量，用于接收调用方法时传入的数据
    return关键字：用于结束方法以及返回方法指定类型的值
    返回值：被return语句返回的值，该值会返回给调用者

重载（overload）

Java允许在一个类中定义多个名称相同的方法，但是参数的类型或个数或顺序必须不同
    重载与返回值无关，如void method(int x)与int method(int y)不是方法重载，不能同时存在
    重载与具体的变量标识符无关，如method(int x)与method(int y)不是方法重载，不能同时存在
    是通过JVM来完成同名方法的调用的，通过参数列表来决定调用的是哪一个方法

参数传递

形式参数：方法定义中参数列表的变量
    实际参数：调用方法时，传入给方法的数值
    调用方法时，传入的数值为基本数据类型（包含String类型），
    复制的是变量的值，形式参数的改变对实际参数不影响
    调用方法时，传入的数值为引用数据类型（String类型除外），
    复制的是变量的地址，形式参数的改变对实际参数有影响
    注意事项
    方法不调用，自己不执行
    方法中不能定义方法， 但是，方法中可以调用方法
    方法定义的位置在类中，其他方法的外面

方法调用方式：

有明确返回值类型的方法调用方式：①单独调用②输出调用③赋值调用
    没有明确返回值类型的方法调用方式：单独调用

构造方法:对象创建时只执行一次的方法、用于给类的成员变量进行初始化操作

定义:修饰符 构造方法名(参数列表){}
特点
构造方法名称必须和类型保持一致。
    一个类中可以有多个构造方法，多个构造方法是以重载的形式存在的
    构造方法是可以被private修饰的，作用：其他程序无法创建该类的对象。
    默认调用已定义的构造方法，没有则自动创建空参构造方法
    super(参数) 与 this(参数) 不能同时在构造方法中存在
构造方法与一般方法区别
构造方法在对象创建时就执行了，而且只执行一次。
    一般方法是在对象创建后，需要使用时才被对象调用，并可以被多次调用
    有了构造方法之后可以对对象的属性进行初始化，那么还需要对应的set和get方法吗？
    需要相应的set和get方法，因为对象在创建之后需要修改和访问相应的属性值时，
    在这时只能通过set或者get方法来操作。

匿名对象

匿名对象是指创建对象时，只有创建对象的语句，却没有把对象地址值赋值给某个变量。
    定义：new 类名([参数]);
    特点：创建匿名对象直接使用，没有变量名。
        匿名对象在没有指定其引用变量时，只能使用一次。
        匿名对象可以作为方法接收的参数、方法返回值使用

内部类：将类（内部类）写在其他类（外部类）的内部，写在其他类的成员位置和局部位置

内部类可以直接访问外部类的成员，包含私有的成员
成员内部类
class 外部类 { 
    修饰符 class 内部类 {//其他代码}
    }
    外部类名.内部类名 变量名 = new 外部类名().new 内部类名();
局部内部类
class 外部类 { 
    修饰符 返回值类型 方法名(参数) {
    class 内部类 {//方法}
    new 内部类().内部类中的方法()；
    }
    }
    在外部类方法中，创建内部类对象，进行访问
实际使用——匿名内部类
临时定义某一指定类型的子类并创建刚刚定义的这个子类的对象
    匿名内部类是创建某个类型子类对象的快捷方式
    new 父类或接口(){//进行方法重写};
    匿名内部类如果不定义变量引用，则也是匿名对象
例子
class Party {//外部类，聚会
    public void puffBall(){// 吹气球方法
        class Ball {// 内部类，气球
            public void puff(){
        System.out.println("气球膨胀了");
    }
    }
    //创建内部类对象，调用puff方法
    new Ball().puff();
    }
    }
    public static void main(String[] args) {
        //创建外部类对象
        Party p = new Party();
        //调用外部类中的puffBall方法
        p.puffBall();
    }

代码块

局部代码块{}：定义在方法或语句中的代码块,用来限制变量的作用范围

构造代码块{}是定义在类中成员位置的代码块

优先于构造方法执行，用于执行所有对象中成员的初始化赋值
    每创建一个对象均会执行一次构造代码块

静态代码块static{}定义在成员位置，使用static修饰的代码块

优于主方法、构造代码块执行，当以任意形式第一次使用到该类时执行
    该类不管创建多少对象，静态代码块只执行一次。
    可用于给静态变量赋值，用来给类进行初始化

成员变量和局部变量区别

定义的位置不同
        定义在类中的变量是成员变量
        定义在方法中或者{}语句里面的变量是局部变量
    在内存中的位置不同
        成员变量存储在堆内存的对象中
        局部变量存储在栈内存的方法中
    声明周期不同
        成员变量随着对象的出现而出现在堆中，随着对象的消失而从堆中消失
        局部变量随着方法的运行而出现在栈中，随着方法的弹栈而消失
    初始化不同
        成员变量因为在堆内存中，所以默认的初始化值
        局部变量没有默认的初始化值，必须手动的给其赋值才可以使用。

包的声明与访问

包

类中声明的包必须与实际class文件所在的文件夹情况相一致，否则，程序运行时会找不到类
    类中包的声明：package 包名.包名.包名…;

    与当前程序不在同一个包中的类（不同文件夹中），要访问的类必须用public修饰才可访问
    带有包的类创建对象：包名.类名 变量名 = new包名.类名();
    导包：import 包名.类名;  import 包名.*;(导入包内所有类)
    在声明包package后，定义所有类class前
    同一个包中的类（同一个文件夹中），或者java.lang包中类时可以省略包名，直接使用该类
public protected default private

常见关键字

this关键字:成员变量名前面加上this.区别成员变量和局部变量

调用本类对象构造方法：this([参数列表]);
    调用本类对象一般方法：方法名([参数列表])或this.方法名([参数列表])
    this到底代表什么呢？this代表的是对象，具体代表哪个对象呢？
    哪个对象调用了this所在的方法，this就代表哪个对象。当创建对象的时候，this存在的
    调用其他构造方法的语句必须定义在构造方法的第一行，原因是初始化动作要最先执行

super关键字

调用父类构造方法：super([实参列表]);
    调用本类中的构造方法：this([实参列表]);
    在创建子类对象时，即构造方法执行时，父类的构造方法会先执行
    因为子类中所有构造方法的第一行有默认的隐式super();语句，访问父类中的空参构造方法

为什么子类对象创建都要访问父类中的构造方法？/子类构造方法为什么会有一句隐式的super()呢？

子类会继承父类中的内容，所以子类在初始化时，必须先到父类中去执行父类的初始化动作
    这样，才可以使用父类中的内容。如果默认的隐式super()语句在父类中没有对应的构造方法，
    那么必须在构造方法中通过this或者super的形式明确要调用的构造方法

如果子类的构造方法第一行写了this调用了本类其他构造方法，那么super调用父类的语句还有吗？

没有，因为this()或者super(),只能定义在构造方法的第一行，因为初始化动作要先执行

父类构造方法中是否有隐式的super呢？

有。构造方法默认第一行都是super();

父类的父类是谁呢？super调用的到底是谁的构造方法呢？

Java体系在设计，定义了一个所有对象的父类Object

final关键字：类、变量或方法不想被继承、修改、重写

final修饰类不可以被继承，但是可以继承其他类。
    final修饰的变量称为常量，这些变量只能赋值一次。
    final修饰的引用类型的变量值为对象地址值，地址值不能更改，
    但是地址内的对象属性值可以修改
    final修饰成员变量必须在创建对象前赋值，否则报错。
    (没有显式赋值时，多个构造方法都要为其赋值)
    final修饰的方法不可以被覆盖,可以继承过来使用，但是可以覆盖其他类的方法

static关键字：不创建对象，就可以调用该类的方法，多个对象共享同一个static成员变量，任何对象对其修改都会使该成员变量值改变

访问静态成员的格式

类名.静态成员变量名
    类名.静态成员方法名(参数)
    静态常量public static final 数据类型 AA_BB = 值;
    接口中的每个成员变量都默认使用public static final修饰
    所有接口中的成员变量已是静态常量，由于接口没有构造方法，所以必须显示赋值。
    可以直接用接口名访问

注意事项

静态内容是优先于对象存在，随类的加载而加载，只能访问静态，不能使用this/super。
    静态修饰的内容存于静态区,同一个类中，静态成员只能访问静态成员
    main方法为静态方法仅为程序执行入口，不属于任何一个对象，可以定义在任意类中
