git 其他命令

查看是否安装了git:git linux安装git:sudo apt-get install git 对当前机器的【所有】git仓库指定不同的用户名和Email地址: $ git config 【–global 】user.name “Your Name” $ git config 【–global 】 user.email “email@example.com“ 在目录下创建git仓库:git init(当前目录下多了一个.git隐藏目录) windows编辑文件不推荐使用自带记事本编辑文件，因为windows在在每个文件开头添加了0xefbbbf（十六进制）的字符，可能导致语法错误，推荐使用NotePad++ 添加文件到git仓库:①git add 添加文件到暂存区 ②git commit -m 将暂存区的文件添加到仓库 查看当前分支的状态:git status 查看文件的修改:git diff 显示git最近到最远的提交commit id历史:git log 显示git最近的命令历史:git reflog 显示git最近一次的commit id历史:git log –pretty=oneline 回退上(上)个版本:git reset –hard HEAD^(^) 回退许多版本:git reset –hard HEAD~100 回退（更新）到指定版本:git reset –hard 每次修改，如果不用git add到暂存区，那就不会加入到commit中。 场景1：当你改乱了工作区某个文件的内容，想直接丢弃工作区的修改时，用命令git checkout – file。 场景2：当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步用命令git reset HEAD ，就回到了场景1，第二步按场景1操作。 git删除文件:git rm git恢复已提交的删除的文件：git checkout –

添加远程仓库:git remote add origin git@github.com:michaelliao/learngit.git 提交代码并同步远程仓库:git push -u origin master 从远程仓库下载代码:git clone git@github.com:michaelliao/gitskills.git

git创建分支并切换到分支:git checkout -b <分支名> 相当于 git branch <分支名> + git checkout <分支名> git查看当前所有分支: git branch git切换分支:git checkout <分支名> （例如master） git合并没有冲突的分支:git merge <分支名> git合并有冲突的分支单没有历史:git merge <分支名> git status 然后手动处理冲突文件，最后git add git commit -m git删除分支:git branch -d <分支名> git查看分支合并图:git log –graph –pretty=oneline –abbrev-commit git合并分支并保存合并历史:git merge –no-ff -m “message” <分支名> git还没提交的时候，创建当前目录的备份:git stash git显示所有备份:git stash list git恢复备份git stash pop，恢复的同时把stash内容也删了 git恢复单个备份不删除stash的内容:git stash apply stash@{0} git强行删除未合并的feature功能分支:git branch -D <分支名> git查看远程仓库信息:git remote [-v] git推送分支:git push origin <分支名> 失败则 git pull 抓取新提交，有冲突额解决冲突并在本地提交，再推送分支， 若提示no tracking information，则说明本地分支和远程分支的链接关系没有创建，用命令git branch –set-upstream-to origin/。建立本地分支和远程分支的关联 git打标签:git tag <标签信息> git根据commit id 打标签并添加标签信息:git tag <标签信息> git打标签指定版本和标签的信息:git tag -a v0.1 -m “version 0.1 released” 1094adb git查看所有标签:git tag git查看指定标签:gti show <标签信息> git删除本地错误的标签:git tag -d git删除远程标签:git push origin :refs/tags/ git推送本地标签到远程:git push origin git一次性把所有未推送的标签推送到远程:git push origin —tags
