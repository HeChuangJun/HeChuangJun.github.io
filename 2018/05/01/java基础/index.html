Java开发环境的搭建

Java概述

Java是一门编程语言是用来编写软件的，主要有以下两种
    互联网：电商、P2P等等；企业级应用：ERP、CRM、BOS、OA

7个常用dos命令(win+R+cmd:编译和运行时java程序使用)

清屏：cls
    切换盘符：例如：C:\ >E:，回车就到了E盘下。一定要回到盘符根目录使用
    进入指定目录：cd；例如D:\>cd develop\Java，这样就进入了D:\develop\Java目录
    返回上一级目录：cd..；0 例如：D:\develop\Java>cd..，回车，就返回到D:\develop目录下
    返回盘符根目录：cd\；
    显示当前目录下的文件和子目录信息：dir；
    运行记事本程序：D:\>  C:\windows\notepad[.exe]

Java开发环境概述（Java程序运行需要的环境）

JDK：Java Development Kit，程序员的电脑上安装JDK；JDK包含JRE
    JRE：Java Runtime Environment，Java运行环境，用户在电脑上安装JRE即可

JDK下载&安装&校验（WIN+R+CMD+JDK安装路径\bin\java.exe）

Java跨平台（操作系统 （Windows，Linux，Mac））特性

Java程序可以在任意操作系统上运行，一次编写到处运行
    实现跨平台需要依赖Java的虚拟机JVM（Java Virtual Machine）

HelloWorld案例

①编写Java源文件（源代码文件），扩展名为.java
        使用最普通的文本编辑器编写，修改文件名称为HelloWorld.java
        注意文件名称的大小写。以及是否已经修改了文件的扩展名
        类名必须与源文件名称相同
        public class HelloWorld {
            public static void main(String[] args) {
                System.out.println("HelloWorld");
            }
        }

    ②编译：通过编译器把源文件编译成字节码文件，扩展名为.class
        编译器：JDK目录下的bin目录javac.exe
        编译命令为：D:\develop\Java\jdk1.7.0_72\bin\javac.exe D:\java\HelloWorld.java
        编译之后会在源文件同目录下生成字节码文件，字节码文件的扩展名为.class
        编译时控制台上不会有输出，如果有输出说明源代码有错误

    ③运行：使用解释器来运行字节码文件
        D:\develop\Java\jdk1.7.0_72\bin\java.exe HelloWord；
        在运行HelloWorld.class文件时不能给出“.class”，切记！！！

    ④配置环境变量（减少输入JDK的bin目录的麻烦）
        在DOS控制台中使用的程序只给出程序名称，而没有给出完整路径时，
        Windows系统会到Path变量保存的路径中查找程序
        鼠标右键点击计算机→属性→高级系统设置→环境变量→系统变量中的Path→编辑
        将JDK安装目录\bin路径配置到PATH变量中，用英文分号与其他变量隔开
        编译：D:/java/>javac HelloWorld.java；运行：D:/java/>java HelloWorld；

注释、关键字与标识符

程序注释：对源码进行解释说明的文字
        单行注释//注释内容
        多行注释/*注释内容*/
        文档注释/**注释内容*/
    关键字：被赋予特殊含义，具有专门用途的均为小写的单词
    标识符
        类名、方法名、变量名、包名、接口名
        组成元素：英文字符: a-zA-Z、数字: 0-9、符号: _与$
        规则：数字不能开头、不可以使用关键字、严格区分大小写，见名知意
        命名规范
            类名&接口名：大驼峰式AaaBbbCcc  
            变量名&方法名：小驼峰式aaaBbbCcc
            常量名：多单词组成是所有字母均大写，使用_连接AAA_BBB_CCC
            包名：多单词组成时所有字母均小写，使用.连接aaa.bbb.ccc
