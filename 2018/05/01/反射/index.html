反射原理

类加载器

类的加载步骤 描述
加载 指用类加载器将class文件读入内存，并为整个class文件创建一个Class对象
（任何类被使用时系统都会建立一个Class对象）
连接 ①验证 是否有正确的内部结构，并和其他类协调一致
②准备 负责为类的静态成员分配内存，并设置默认初始化值
③解析 将类的二进制数据中的符号引用替换为直接引用
初始化
类加载到内存的时机
创建类的实例
初始化某个类的子类
类的静态变量，或者为静态变量赋值
类的静态方法
使用反射方式来强制创建某个类或接口对应的java.lang.Class对象
直接使用java.exe命令来运行某个主类
类加载器分类(超类ClassLoader) 作用 位置
BootstrapClassLoader引导类加载器 Java核心类的加载(System,String) JRE/lib/rt.jar
ExtensionClassLoader扩展类加载器 JRE的扩展目录jar包的加载 JRE/lib/ext/*.jar
AppClassLoader应用类加载器 三方jar包和自己编写的java文件
Class对象常用ApI 描述
Class对象.getClassLoader(); 获得Class对象的类加载器
classLoader.getResource(“相对src路径”).getPath(); 获得classes(src)下的任何资源绝对路径
classLoader.getResourceAsStream(“”); 获得classes下(src)下的任何资源inputStream

反射的使用

反射对象 方法
获得Class对象 类名 变量名 = new 类名();Class c1 = 变量名.getClass();
Class c2 = 类名.class;(常用)
Class c3 = Class.forName(“完整类名”);(常用)
通过Class对象获得Constructor类 Constructor getConstructor(Class<?>… parameterTypes);(public)
Constructor getDeclaredConstructor(Class<?>… parameterTypes);(All)
Constructor<?>[] getConstructors()(public)
Constructor<?>[] getDeclaredConstructors(All))
通过Constructor类创建对象 T newInstance(Object… initargs)(传递构造函数参数)
通过Class对象获得Field类 Field getField(String name);(本类及父类public)
Field getDeclaredField() (本类All)
Field[] getFields() (本类及父类public)
Field[] getDeclaredFields() ;(本类All)
通过class对象获得Method类 Method getMethod(String name, Class<?>… parameterTypes)(本类及父类public)
Method getDeclaredMethod(String name, Class<?>… parameterTypes)(本类All)
Method[] getMethods() (本类及父类public)
Method[] getDeclaredMethods() (本类All)
取消Constructor、Field、Method父类 void setAccessible(boolean flag) throws SecurityException(true取消)
AccessibleObject语言访问检查(允许访问私有)
利用Field类对属性进行赋值取值 void set(Object obj,Object value)(obj传入实例化对象,静态属性则为null)
Object get(Object obj)(obj传入实例化对象,静态属性则为null)
利用Method类执行成员方法 Object invoke(Object obj,Object… args)(obj传入实例化对象,静态方法则为null)
