反射原理

类的加载步骤

加载  指用类加载器将class文件读入内存，并为整个class文件创建一个Class对象
    （任何类被使用时系统都会建立一个Class对象）
    连接 验证 是否有正确的内部结构，并和其他类协调一致
        准备 负责为类的静态成员分配内存，并设置默认初始化值
        解析 将类的二进制数据中的符号引用替换为直接引用
    初始化

类加载到内存的时机

创建类的实例
    初始化某个类的子类
    类的静态变量，或者为静态变量赋值
    类的静态方法
    使用反射方式来强制创建某个类或接口对应的java.lang.Class对象
    直接使用java.exe命令来运行某个主类

类加载器（分类）负责将.class文件加载到内存中，并为之生成对应的Class对象

Bootstrap ClassLoader 根类加载器 /引导类加载器，
    Java核心类的加载 比如System,String等。在JDK中JRE的lib目录下rt.jar文件中
    Extension ClassLoader 扩展类加载器 
    JRE的扩展目录中jar包的加载。在JDK中JRE的lib目录下ext目录
    System ClassLoader 系统类加载器
    JVM启动时加载来自java命令的class文件、classpath环境变量所指定的jar包和类路径

反射机制

JAVA反射机制是在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；
    对于任意一个对象，都能够调用它的任意一个方法和属性；
    这种动态获取的信息以及动态调用对象的方法的功能称为java语言的反射机制。
    自己的理解
    类加载进入内存后产生.class字节码文件、该类的对象、.class字节码文件对应的class对象
    获得class对象并利用其方法获得class文件所有的属性和方法然后就能执行创建对象，
    调用构造方法，一般方法，设置属性值等操作，不用修改源码就能解决动态调用方法

反射的使用

①获得class类对象对字节码文件的描述对象（有且只有一个）不能创建，只能获得

Class对象无公共构造方法,加载类时由Java虚拟机和调用类加载器defineClass方法自动构造
    通过Object类中的getObject()方法,必须明确Person类型.
    Person p = new Person();   Class c = p.getClass();
    通过 类名.class 获取到字节码文件对象（任意数据类型都具备一个class静态属性）
    Class c2 = Person.class;        必须明确Person类型.
    通过Class类中的方法（将完整类名作为字符串传递给Class类中的静态方法forName即可）
    Class c3 = Class.forName("cn.junye.Person");
    指定这种类型的字符串就行.扩展更强.不需要知道类.提供字符串,按照配置文件加载就行

②class类获得构造方法的方法及其返回封装好的Constructor类

返回一个构造方法
    public Constructor<T> getConstructor(Class<?>... parameterTypes) 
    获取public修饰, 指定参数类型所对应的构造方法Xxx.class/null
    public Constructor<T> getDeclaredConstructor(Class<?>... parameterTypes)
    获取指定参数类型所对应的构造方法(包含私有的)
    返回多个构造方法
    public Constructor<?>[] getConstructors()获取所有public 修饰的构造方法
    public Constructor<?>[] getDeclaredConstructors()获取所有构造方法(包含私有的)

③利用Constructor类创建对象的方法

public T newInstance(Object... initargs)//传递实际参数/null

②class类获得成员变量的方法及其返回封装好的Field类

返回一个成员变量
    public Field getField(String name) 获取指定的 public修饰的变量
    public Field getDeclaredField(String name) 获取指定的任意变量
    返回多个成员变量
    public Field[] getFields() 获取所有public 修饰的变量
    public Field[] getDeclaredFields() 获取所有的 变量 (包含私有)

③利用Field类对属性进行赋值取值的方法（必须创建对象以传入方法）

public void set(Object obj, Object value)给指定对象的成员变量设置值
    public Object get(Object obj)返回指定对象的成员变量值

②class类获得成员方法的方法及其返回封装好的Method类

返回获取一个方法：
    public Method getMethod(String name, Class<?>... parameterTypes)
    获取public 修饰的方法
    public Method getDeclaredMethod(String name, Class<?>... parameterTypes)
    获取任意的方法，包含私有的
    参数1: name 要查找的方法名称； 参数2： parameterTypes 该方法的参数类型
    返回获取多个方法：
    public Method[] getMethods() 获取本类与父类中所有public 修饰的方法
    public Method[] getDeclaredMethods() 获取本类中所有的方法(包含私有的)

③利用Method类对象执行class文件中的成员方法相应的方法（必须创建对象以传入方法）

public Object invoke(Object obj,  Object... args) 
    执行指定对象obj中，当前Method对象所代表的方法，方法要传入的参数通过args指定

访问class内容时自动调用访问控制语言检查的方法

AccessibleObject 是Field类、Method类、Constructor类的父类
    暴力访问私有构造方法、一般方法、成员变量，在访问前设置，true为取消Java语言访问检查
    public void setAccessible(boolean flag) throws SecurityException

注解

符合一定格式的语法 @xxxx，代码级别的说明，与类、接口、枚举同一层次
    注释：在阅读程序时清楚----给程序员看的
    注解：给jvm看的，给机器看的
    主要用于代替配置文件
    编译检查、代码分析、编写文档

关于配置文件与注解开发的优缺点：

注解优点：开发效率高 成本低  
    注解缺点：耦合性大 并且不利于后期维护

常用的注解

@Override：告知编译器此方法是覆盖父类的
    @Deprecated：标注过时①安全问题②新的API取代
    @SuppressWarnings（value）：压制警告
    deprecation：忽略过时
    rawtype：忽略类型安全
    unused：忽略不使用
    unchecked：忽略安全检查
    null：忽略空指针
    all：忽略所有
    不同的注解只能在不同的位置使用(方法上、字段上、类上)

自定义注解

编写一个注解

@Retention(xxx)
    @Target(xxx)
    public @interface Myanno{
    String value();
    int age() default 28;
    }
注解的属性：
语法：修饰符 返回值类型 属性名称()[default 值]; 修饰符默认public abstract
    注意：如果属性的名字是value，并且注解的属性值有一个 那么在使用注解时可以省略value
注解属性返回值类型
1.基本类型
    2.String
    3.枚举类型
    4.注解类型
    5.Class类型
    6.以上类型的一维数组类型
元注解：代表修饰注解的注解，限制定义的注解的特性,被元注解修饰过的注解才能被使用
@Retention(xxx)
RetentionPolicy.SOURCE:注解在源码级别可见
    RetentionPolicy.CLASS：注解在字节码文件级别可见
    RetentionPolicy.RUNTIME：注解在整个运行阶段都可见
    要想解析使用了注解的类 ， 那么该注解的Retention必须设置成Runtime
@Target(xxx)
代表注解修饰的范围：类上使用，方法上使用，字段上使用
    Element.FIELD:字段上可用此注解
    Element.METHOD:方法上可以用此注解
    Element.TYPE:类/接口上可以使用此注解
    Element.CONSTRUCTOR:构造方法上面可以用

解析使用了注解的类(实质：从注解中解析出属性值)

字节码对象存在获得注解相关的方法
    boolean isAnnotationPresent(Class<? extends Annotation> annotationClass) ：
    判断该字节码对象身上是否使用该注解了
    T getAnnotation(Class<A> annotationClass) ：
    获得该字节码对象身上的注解对象

使用注解:在类/方法/字段 上面是@XXX

注解可以没有属性，如果有属性要用()如Myanno或Myanno()
    属性格式：属性名=属性值，多个属性使用，分隔如Myanno(username="rose")
    如果属性名为value，且当前只有一个属性，value可以省略
    使用多个属性时，k的名称为value不能省略
    如果属性类型为数组，格式为{1,2,3}如arrs={"a""b"}
    如果属性类型为数组，值只有一个{}可以省略如arrs=a
    一个对象上，注解只能使用一次，不能重复使用

动态代理

动态代理的产生

动态代理概念图解

动态代理—-aop

JDK的Proxy方式实现的动态代理 目标对象必须有接口 没有接口不能实现动态代理
    动态代理：不用手动编写一个代理对象，不需要一一编写与目标对象相同的方法，
    这个过程，在运行时 的内存中动态生成代理对象。------字节码对象级别的代理对象
    static Object newProxyInstance
    (ClassLoader loader, Class<?>[] interfaces, InvocationHandler h) 
    Object就是代理对象
    loader：代表与目标对象相同的类加载器:UserServiceImpl.getClass().getClassLoader()
    Class<?>[] interfaces：代表与目标对象实现的所有的接口字节码对象数组
    目标对象类实例.getClass().getInterfaces();
    new Class[]{UserService.class}
    h：具体的代理的操作，InvocationHandler接口
    接口方法Object invoke(Object proxy, Method method, Object[] args)
    proxy：代理对象，
    method：代理对象当前执行方法method.getName()\method.invoke(对象，实际参数)
    args：方法实际参数

动态代理的使用

目标接口

public interface TargetInterface {
    public void method1();
    public String method2();
    public int method3(int x);
    }

目标接口实现类

public class Target implements TargetInterface{
    @Override
    public void method1() {
    System.out.println("method1 running...");
    }
    @Override
    public String method2() {
    System.out.println("method2 running...");
    return "method2";
    }
    @Override
    public int method3(int x) {
    return x;
    }
    }

动态代理

方式1
public static void main(String[] args) {
    final Target target = new Target();
    TargetInterface proxy = (TargetInterface) Proxy.newProxyInstance(
    target.getClass().getClassLoader(), 
    target.getClass().getInterfaces(), 
    new InvocationHandler() {
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) 
    throws Throwable {
    Object invoke = method.invoke(target, args);//目标对象的相应方法
    return invoke;
    }
    }
    );
    proxy.method1();//调用invoke目标对象method1方法 args：null 返回值null
    //调用invoke目标对象method2方法 args:null 返回值method2
    String method2 = proxy.method2();
    //调用invoke目标对象method3方法 args:Object[]{100} 返回值100
    int method3 = proxy.method3(100);
    }
方式2
public class ProxyTest {
    @Test
    public void test1(){
    //获得动态的代理对象----在运行时 在内存中动态的为Target创建一个虚拟的代理对象
    //objProxy是代理对象 根据参数确定到底是谁的代理对象
    TargetInterface objProxy = (TargetInterface) Proxy.newProxyInstance(
    Target.class.getClassLoader(), //与目标对象相同的类加载器
    new Class[]{TargetInterface.class}, //目标对象接口字节码数组
    new InvocationHandler() {
    //invoke 代表的是执行代理对象的方法
    @Override
    //method：代表目标对象的方法字节码对象
    //args:代表目标对象的响应的方法的参数
    //被执行几次？看代理对象调用方法几次、代理对象调用接口相应方法 都是调用invoke方法
    public Object invoke(Object proxy, Method method, Object[] args) 
    throws Throwable {
    System.out.println("目标方法前的逻辑");
    //执行目标对象的方法，并返回目标对象方法执行后的结果
    Object invoke = method.invoke(new Target(), args);
    System.out.println("目标方法后的逻辑");
    //retrun返回的值给代理对象
    return invoke;
    }
    }
    );
    objProxy.method1();String method2 = objProxy.method2();
    System.out.println(method2);
    }
    }
