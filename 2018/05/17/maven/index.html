    maven项目管理工具

    maven概述

    apche下的项目管理工具,开源java项目
①一步构建项目：项目构建过程标准化，一个命令完成、编译、测试、运行、打包、部署、运行(tomcat:run)
②依赖管理：不用手动导入jar包，在pom.xml定义坐标自动从maven仓库中下载，不易出错
③跨平台 window 和 linux都可以使用 
④遵循规范，提高开发效率，降低项目维护成本

    普通项目与maven项目构建过程

    普通项目构建过程
①eclipse中创建一个java-web工程
②在工程中编写源代码及配置文件等
③对源代码进行编译，java文件编译成class文件
④执行junit测试
⑤将工程打成war包部署在tomcat中运行

maven项目构建过程
清理(clean)、编译(commpile)、测试、报告、
打包(package：java→jar、web→war)、部署
运行mavenLtomcat：run、tomcat7:run
一个命令完成构建、运行、方便快捷，每个构建阶段规范，有利于团队开发

    传统项目和普通项目依赖管理

    传统项目依赖管理：手动拷贝jar包到WEB-INF目录下。jar包版本冲突、jar包难找、jar包过大
maven项目依赖管理：通过pom.xml对jar包版本同一管理，避免冲突、可以从maven仓库中下载jar包

    maven仓库

    本地仓库(默认:${usr.dir}/.m2/repository)
优先查找本地仓库、没有查远程(可以私服)、最后中央、查到自动下载到本地仓库保存
远程仓库：互联网或局域网内
中央仓库：http：//repo1.maven.org/maven2、maven团队自己维护
配置本地仓库
F:\apache-maven-3.5.3\conf\setting.xml
<localRepository>F:\repository</localRepository>

    maven工程目录

    常用maven命令

    mvn compile 将src/main/java下的 文件编译为class文件 并放到target/classes目录下,不编译测试的代码
mvn test 执行src/test/java的单元测试类
mvn clean 删除target目录的内容(包括打包结果和编译结果)，清理后编译
mvn package java工程→jar、web工程→war包，默认jar包名称：artificild-version.jar
mvn install：将maven打成jar包或者war包发布到本地仓库(模块开发的时候使用)安装到仓库/groupId/artifacid/version
tomcat:run 一键启动

    maven概念模型内容

    ①项目对象模型
pom.xml定义项目坐标、依赖、信息、插件目标

②标准集合
标准目录结构、生命周期阶段、依赖管理有标准的坐标定义

③项目生命周期
清理(clean)、编译(commpile)、测试(test)、报告、打包(package：java→jar、web→war)、部署 通过命令实现生命周期

④依赖管理系统：
对jar包同一管理

⑤运行定义在生命周期阶段中插件的目标的逻辑
maven管理项目生命周期过程都是基于插件完成的

    创建maven项目一般步骤

    创建maven项目要跳过骨架、不跳过这目录不全、web选择war

    在src/webapp/WEB-INF/web.xml配置web.xml，不配置则报错
定义maven坐标(maven对jar包的身份定义)eclipse自动生成
<modelVersion>4.0.0</modelVersion>
    <groupId>cn.itcast.maven</groupId>项目名称：组织名+项目名。类似包名
    <artifactId>maven-helloworld</artifactId>模块名
    <version>0.0.1-SNAPSHOT</version>当前版本号snapshot快照版本非正式版本，release为正式发布版
    <packaging>war</packaging>打包类型jar、war、pom(父工程)使用这个设置与package有关
    <name>第一个maven工程</name>
    <description>第一个maven工程</description>
定义插件(编译器)jdk1.8(设置编译版本)
<build>
    <!-- 配置了很多插件 -->
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.5.1</version>  
            <configuration>
            <source>1.7</source>
            <target>1.7</target>
            <encoding>UTF-8</encoding>
            </configuration>
        </plugin>
    </plugins>
</build>
定义依赖的jar包在pom.xml

    配置其他应该配置及编写servlet，jsp等、运行tomcat：run

    pom.xml基本配置

    <project>:文件的根节点
    <modelVersion>:pom.xml使用的对象模型版本
    <groupId>:项目名称：组织名+项目名。类似包名
    <artifactId>:模块名，子项目名或模块名称
    <version>:当前版本号snapshot快照版本非正式版本release为正式发布版
    <packaging>:打包类型jar、war、pom父工程使用这个设置与package有关
    <name>:项目的显示名，常用于maven的生成文档
    <description>:项目描述，常用于maven的生成文档
    <dependencies>:项目依赖构件配置，配置项目依赖构件的坐标
<build>:项目构建配置，配置编译，运行插件等

    依赖管理

    查找坐标
方法一从互联网搜索
http://search.maven.org/、http://mvnrepository.com/
方法二使用maven插件的索引功能
如果在本地仓库看jar包，可以在pom.xml中右键添加依赖
右击项目→maven→add dependency

依赖范围(添加依赖jar包格式)
依赖war包、依赖模块\依赖范围由强到弱的顺序是：compile>provided>runtime>test

    依赖范围对传递依赖的影响

    依赖传递与依赖冲突解决

    ①调解原则
第一声明者优先原则：谁先声明谁先依赖
路径近者优先原则：谁的依赖层小，谁先被依赖
②排除依赖
<dependency>
    <groupId>org.apache.struts</groupId>
    <artifactId>struts2-spring-plugin</artifactId>
    <version>2.3.24</version>
    <exclusions>
        <exclusion>
            <groupId>org.springframework</groupId>
            <artifactId>spring-beans</artifactId>
        </exclusion>
    </exclusions>
</dependency>
③锁定版本
配置了锁定版本必须相应地配置依赖，但不需要指定版本
<dependencyManagement>
    <dependencies>
        <dependency>
        <groupId>org.springframework</groupId>
        <artifactId>spring-bean</artifactId>
        <version>4.2.4.RELEASE</version>
        </dependency>
    </dependencies>
</dependencyManagement>

    分模块开发(重点)

    继承
避免了在三层架构中重复定义pom.xml的内容，
将这些配置提取出来在父工程中配置
聚合
分模块开发完成后会打一个war运行、将每个模块合在一起运行
示意图

    步骤(都要跳过骨架)
①创建父工程maven Project
父工程packaging方式为pom、定义好pom.xml
    不添加junit依赖、install发布到仓库方便自工程继承
②创建maven Module子工程dao
packaging方式为jar、
    parent Project 选择父工程、模块中pom.xml父工程的信息    
    <project xmlns="http://maven-apache.org/POM/4.0.0"....>
    <modelVersion>4.0.0</modelVersion>
    <parent>
    <groupId>cn.itcast.maven</groupId>
    <artifactId>maven-parent</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <artifactId>ssh-dao</artifactId>
    </parent>
    <project>
③创建maven Module子工程service
packaging方式为jar、
    parent Project 选择父工程、同时配置dao依赖模块中pom.xml父工程的信息
    <dependency>
    <groupId>cn.itcast.ssh-maven</groupId>
    <artifactId>ssh-dao</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    </dependency>
④创建maven Module子工程web
packaging方式为war、
    parent Project 选择父工程、同时配置service依赖模块中pom.xml父工程的信息
    <dependency>
    <groupId>cn.itcast.ssh-maven</groupId>
    <artifactId>ssh-service</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    </dependency>
模块聚合：修改父工程的pom.xml
<modules>
    <module>ssh-dao</module>
    <module>ssh-service</module>
    <module>ssh-web</module>
</modules>
