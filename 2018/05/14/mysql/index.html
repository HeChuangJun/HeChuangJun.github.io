分类 说明 作用 举例
DML 数据库操作语言 对数据库中标的记录进行更新 insert、delete、update
DDL 数据库定义语言 定义数据库对象 create、alter、drop等
DQL 数据库查询语言 查询数据库中表的记录 select
DCL 数据库控制语言 定义数据库的访问权限和安全级别 grant等

mysql事务

事务类型 说明 详细
默认事务 一条sql语句就是一个事务 默认就开启事务并提交事务
手动事务 开启事务 start transaction
事务提交 commit从开启事务到事务提交，中间所有的sql都认为有效\真正更新数据库
事务回滚 rollback从开启事务到事务回滚，中间所有的sql操作都认为无效数据库被更新
JDBC事务 默认是自动事务 执行sql语句：executeUpdate()每执行一次executeUpdate方法 代表事务自动提交
JDBC手动事务 开启事务 conn.setAutoComnmit(false);
提交事务 conn.commit();
回滚事务 conn.rollback();
执行sql的connection与开启事务的 connnection必须是同一个才能对事务进行控制

ThreadLocal：实现的是通过线程绑定的方式传递参数

事务特性 说明
原子性（Atomicity） 事务是一个不可分割的工作单位，事务中的操作要么都发生，要么都不发生
隔离性（Isolation） 指多个用户并发访问数据库时，一个用户的事务不能被其它用户的事务所干扰，多个并发事务之间数据要相互隔离
一致性（Consistency） 一个事务中，事务前后数据的完整性必须保持一致。
持久性（Durability） 事务一旦被提交，它对数据库中数据的改变就是永久性的，接下来即使数据库发生故障也不应该对其有任何影响。
并发访问问题 说明
脏读 B事务读取到了A事务尚未提交的数据-要求B事务要读取A事务提交的数据
不可重复读 一个事务中 两次读取的数据的内容不一致-unpdate
幻读/虚读 一个事务中 两次读取的数据的数量不一致-insert、delete
事务隔离级别 说明
read uncommitted 读取尚未提交的数据 ：哪个问题都不能解决
read committed 读取已经提交的数据 ：可以解决脏读 —- oracle默认的
repeatable read 重读读取：可以解决脏读 和 不可重复读 —mysql默认的
serializable 串行化：可以解决 脏读 不可重复读 和 虚读—相当于锁表
隔离级别的性能 read uncommitted>read committed>repeatable read>serialazable
安全性 read uncommitted<read committed<repeatable read<serialazable
