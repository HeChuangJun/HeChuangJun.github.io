事务：

一件事情有n个组成单元 要不这n个组成单元同时成功 要不n个单元就同时失败
    保证一个事务中多次操作要么全部成功，要么全部失败

mysql事务

默认的事务

一条sql语句就是一个事务 默认就开启事务并提交事务

手动事务：

显示的开启一个事务：start transaction
    事务提交：commit从开启事务到事务提交，中间所有的sql都认为有效\真正更新数据库
    事务回滚：rollback从开启事务到事务回滚，中间所有的sql操作都认为无效数据库被更新

事务特性

原子性（Atomicity）：

事务是一个不可分割的工作单位，事务中的操作要么都发生，要么都不发生。

隔离性（Isolation）

指多个用户并发访问数据库时，一个用户的事务不能被其它用户的事务所干扰，
    多个并发事务之间数据要相互隔离。

一致性（Consistency）

一个事务中，事务前后数据的完整性必须保持一致。

持久性（Durability）

事务一旦被提交，它对数据库中数据的改变就是永久性的，
    接下来即使数据库发生故障也不应该对其有任何影响。

并发访问问题

如果不考虑隔离性，事务存在三种并发访问问题。
    脏读：B事务读取到了A事务尚未提交的数据-要求B事务要读取A事务提交的数据
    不可重复读：一个事务中 两次读取的数据的内容不一致-unpdate
    幻读/虚读：一个事务中 两次读取的数据的数量不一致-insert、delete

事务的隔离级别

read uncommitted : 读取尚未提交的数据 ：哪个问题都不能解决
    read committed：读取已经提交的数据 ：可以解决脏读 ---- oracle默认的
    repeatable read：重读读取：可以解决脏读 和 不可重复读 ---mysql默认的
    serializable：串行化：可以解决 脏读 不可重复读 和 虚读---相当于锁表
    隔离级别的性能：
    read uncommitted>read committed>repeatable read>serialazable 
    安全性：
    read uncommitted<read committed<repeatable read<serialazable

说说delete与truncate的区别？

delete删除的时候是一条一条的删除记录，它配合事务，可以将删除的数据找回。
    主键不重置，如果设置了auto_increment记录数
    truncate删除，它是将整个表摧毁，然后再创建一张一模一样的表。
    它删除的数据无法找回。主键重置

修改root密码

方法一:

在my.ini的[mysqld]字段加入：skip-grant-tables
    重启mysql服务，这时的mysql不需要密码即可登录数据库 然后进入mysql
    mysql>use mysql;
    mysql>更新 user set password=password('新密码') WHERE User='root';
    mysql>flush privileges;
    运行之后最后去掉my.ini中的skip-grant-tables，重启mysqld即可

方法二:

不使用修改my.ini重启服务的方法，通过非服务方式
    加skip-grant-tables运行mysql来修改mysql密码
    停止mysql服务
    打开命令行窗口，在bin目录下使用mysqld-nt.exe启动，
    在命令行窗口执行: mysqld-nt --skip-grant-tables
    然后另外打开一个命令行窗口，登录mysql，此时无需输入mysql密码即可进入。
    按以上方法修改好密码后,关闭命令行运行mysql的那个窗口，
    此时即关闭了mysql，如果发现mysql仍在运行的话可以结束掉对应进程来关闭。
    启动mysql服务
