jsp

建立在Servlet规范上的动态网页开发
    产生原因:html页面输出太麻烦

脚本

<%java代码%>JSPScriptlets：被翻译到Servlet中的service方法的局部变量。
    调用完则释放内存空间，与输出内容有关的用
    <%=java变量或表达式%>JSP表达式：被翻译成servlet类中service方法内部out.print()，直接输出
    <%!java代码%>JSP声明语句会被翻译成servlet类的成员的内容，
    定义的是成员方法/变量，静态方法/变量,对整个jsp页面有效

注释

Html注释：<!--注释内容-->--可见范围jsp源码、翻译后的servlet、页面显示html源码
    java注释：//单行注释/*多行注释*/--可见范围jsp源码翻译后的servlet
    jsp注释：<%--注释内容--%>-----可见范围jsp源码可见

指令:指导jsp翻译和运行的命令

<%@page language=”java” contentType=”text/html;charset=UTF-8”pageEncoding=”UTF-8”% >

属性
    language：jsp脚本中可以嵌入的语言种类
    pageEncoding：当前jsp文件的本身编码---内部可以包含contentType
    contentType：response.setContentType(text/html;charset=UTF-8)
    session：是否jsp在翻译时自动创建session
    session="true"
    import：导入java的包 可多次，其他职能一次
    <%@page import="java.util.*"%>
    errorPage：当当前页面出错后跳转到哪个页面
    isErrorPage：当前页面是一个处理错误的页面
    isErrorPage="true"
    （属性最多，根据不同属性指导整个页面特性）

<%@taglib uri=”标签库地址” prefix=”前缀”%> 使用：<前缀:xxx>

在jsp页面中引入标签库（jstl标签库、struts2标签库）

<%@include file=”被包含的文件地址，相对于webcontent地址”%> include（静态包含）

pageContext

pageContext:指当前页面转换后的Servlet类的实例、jsp页面的上下文对象
    获得其他8大隐式对象例如： pageContext.getRequest()
    域对象,向指定的其他域中存取数据
    setAttribute(String name,Object obj,int scope)
    getAttribute(String name,int scope)
    removeAttrbute(String name)
    removeAttrbute(String name,int scope)
    findAttribute(String name)
    依次从page域，request域，session域，application域中获取属性，
    在某个域中获取后将不在向后寻找，都没有则返回null
    本身:pageContext.setAttribute("name", "sunba");

out:用于页面输出out的类型：JspWriter

out作用就是想客户端输出内容----out.write()
    out缓冲区默认8kb 可以在page属性中配置buffer=“0kb” 代表关闭out缓冲区 
    内容直接写到respons缓冲区，没有则先将response.getWriter（）.write（“xx”），
    先打出来，然后再打out.write的，其他内容按顺序输出

四大作用域

page域：当前jsp页面范围
    request域：一次请求
    session域：一次会话
    application域：整个web应用
    获得修改指定作用域的数据
    例：pageContext.setAttrbute("name","pvalue")
    例：pageContext.getAttrbute("name")
    所有对象都可以存取数据request.setAttribute("age", 30);？？？？

jsp标签/动作

<jsp:include page="" flush=“true/false”/> 动态包含,先编译被包含的页面,然后编译包含页面
    flush是否将输出内容刷新到客户端，默认false
    请求转发：<jsp:forward page="要转发的资源相对于webcontent地址"></jsp:forward>
    静/动态包含区别：静态包含翻译后合在一起，动态包含用include方法将两个页面包在一起

运行原理

本质就是servlet，第一次被访问的时候被web容器翻译成servlet再执行
    访问helloServlet.jsp->helloServlet_jsp.java源文件->helloServlet_jsp.class字节码文件
    P被翻译后的servlet在Tomcat的work目录中可以找到
    F:\Tomcat\apache-tomcat-7.0.52\work\Catalina\localhost\WEB17\org\apache\jsp
    查看*.jsp文jsp继承了HttpJspBase，该类又继承了HttpServlet，故jsp就是servlet

el:Expression Language

获得四大域中数据${}，

自定义的数据必须放在域中el才能得到
    获得指定域中的值
    获得page域中的值：${pageScope.key};
    获得request域中的值：${requestScope.key};
    获得session域中的值：${sessionScope.key};
    获得application域中的值：${applicationScope.key};
    寻找全域的值
    从四个域中获得某个值$(key);字符串、数组、list、map<string,string>、map<String,javaBean>
    依次从page域，request域，session域，application域中获取属性，
    在某个域中获取后将不在向后寻找，如果都没有则返回null

11个内置对象

pageScope,requestScope,sessionScope,applicationScope ---- 获取JSP中域中的数据
    param,paramValues - 接收参数.
    header,headerValues- 获取请求头信息
    initParam- 获取全局初始化参数
    cookie- WEB开发中cookie
    pageContext- WEB开发中的pageContext.
    使用${内置对象.key}

执行表达式

pageContext获得其他八大对象${pageContext.request.contextPath}
    $(pageContext.request.contextPath)/XXX
    相当于<%=pageContext.getRequest().getContextPath%>获得WEB应用的名称
    <!-- el可以执行表达式运算 -->
    ${1+1 }
    ${1==1?true:false }
    <!-- empty 判定某个对象是否是null  是null返回true -->
    ${empty list}

jstl（JSP Standard Tag Library)

JSP标准标签库，可以嵌入在jsp页面中使用标签的形式完成业务逻辑等功能。
    jstl出现的目的同el一样也是要提到jsp页面中的脚本代码

使用

①导入jar包jstl.jar和standard.jar
    jstl.jar文件包含JSTL规范中定义的接口和相关类
    standard.jar文件包含用于实现JSTL的.class文件以及JSTL中5个标签库描述符文件（TLD）
    ②引入<%@taglib uri="http://java.sun.com/jsp/jstl/core"prefix="c"%>
    <c:forEach begin="0"end="5"var="i" step=“2”>${i}<br/></c:forEach>
    <c:forEach items="将要迭代的对象" var="每一个对象的值" 【varStatus=""】>
    获得各种数据结构的值
    <c:forEach items="${strList }" var="str">${str }<br/></c:forEach>    
    <c:forEach items="${userList}" var="user">
        user的name：${user.name }------user的password：${user.password }<br/>
    </c:forEach>
    <c:forEach items="${strMap }" var="entry">
        ${entry.key }====${entry.value }<br/>
    </c:forEach>    
    <c:forEach items="${userMap }" var="entry">
        ${entry.key }:${entry.value.name }--${entry.value.password }<br/>
    </c:forEach>
    <c:if test="">test设置判断条件

listener

监听某个对象的的状态变化的组件，当被监听对象发生改变时，立即调用相应的方法处理
    事件源：被监听的对象  ----- 三个域对象 request  session  servletContext
    监听器：监听事件源对象  事件源对象的状态的变化都会触发监听器 ---- 6+2
    注册监听器：将监听器与事件源进行绑定
    响应行为：监听器监听到事件源的状态变化时 所涉及的功能代码 ---- 程序员编写代    码
    事件：事件源行为

域对象属性的方法

setAttribute(name,value)--- 触发添加、修改属性的监听器的方法   
    getAttribute(name)
    removeAttribute(name)  --- 触发删除属性的监听器的方法

3个与域对象创建与销毁

ServletContextListener
①初始化：对象、数据、加载数据库驱动，连接池的初始化
    ②加载一些初始化的配置文件 --- spring的配置文件
    ③任务调度----定时器----Timer/TimerTask
    ServletContext的生命周期
    何时创建：服务器启动创建
    何时销毁：服务器关闭销毁
    任务调度
HttpSessionListener
HttpSession对象的生命周期
    何时创建：第一次调用request.getSession时创建
    何时销毁：服务器关闭销毁  session过期  手动销毁
    sessionCreated、sessionDestroyed
ServletRequestListener
ServletRequest的生命周期
    创建：每一次请求都会创建request
    销毁：请求结束
    requestDestroyed、requestInitialized
3个与域对象的属性变化有关
若是改属性的话，只能获得之前被改的值，而不是改好的值
    ServletContextAttributeLIstener
    HttpSessionAttributeListener
    ServletRequestAttributeListener

使用

①编写监听器类去实现监听器接口
    ②覆盖监听器的方法ContextInitialized和contextDestroyed
    ③在web.xml中进行配置---注册  
    <listener><listener-class>XXXXListener</listener-class></listener>

2个与session域中存储对象相关

绑定与解绑HttpSessionBindingListener(给对象使用）

绑定状态：就一个对象被放到session域中(setAttrbute())
    解绑状态：就是这个对象从session域中移除了remove
    ①被绑对象javaBean实现HttpSessionBindingListener接口，不需web.xml注册
    ②实现接口的所有方法

钝化与活化HttpSessionActivationListener

钝化状态：将session内存中的对象持久化（序列化）到磁盘（作用重要）服务器关闭
    活化状态：就是将磁盘上的对象再次恢复到session内存中
    ①javaBean实现HttpSessionActivationListener,Serializable并覆盖重写所有方法
    ②在META-INF创建context.xml文件
    <Context>
    maxIdleSwap:session中的对象多少分钟不使用就钝化，
    directory:钝化后的对象的文件写到磁盘的哪个目录下，
    配置钝化的对象文件在work/catalina/localhost/钝化文件名  test-->
    <Manager className="org.apache.catalina.session.PersistentManager" maxIdleSwap="1">
    <Store className="org.apache.catalina.session.FileStore" directory="test" />
    </Manager>
    </Context>

即将要被绑定到session中的对象有几种状态

绑定状态：就一个对象被放到session域中
    解绑状态：就是这个对象从session域中移除了
    钝化状态：是将session内存中的对象持久化（序列化）到磁盘
    活化状态：就是将磁盘上的对象再次恢复到session内存中

filter

filter是对客户端访问资源的过滤，符合条件放行，不符合条件不放行，
    并且可以对目标资源访问前后进行逻辑处理，运行在服务器端的程序，
    先于与之相关的Servlet或jsp前运行，对请求资源过滤的功能,可以处理请求和响应
    ①编写一个过滤器的类实现Filter接口
    ②实现接口中尚未实现的方法(着重实现doFilter方法)
        ①init(Filterconfig)：filter对象初始化方法 filter对象创建时执行，与服务器启动相近，
        config代表该Filter对象的配置信息的对象，内部封装是该filter的配置信息。servletconfig
        //获得web.xml中filter 的名称<filter-name>QuickFilter1</filter-name>
        System.out.println(filterConfig.getFilterName());
        System.out.println(filterConfig.getInitParameter("aaa"));//获得filter初始化参数
        System.out.println(filterConfig.getInitParameterNames());
        filterConfig.getServletContext();    //获得servletContext
        ②doFilter(ServletRequest,ServletResponse,FilterChain)：
        filter执行过滤的核心方法，若资源在已经被配置到这个filter进行过滤的话，
        则每次访问这个资源都会执行该方法
        FilterChain：过滤器链对象，放行请求chain.doFilter(request, response);
        ③destory()：filter销毁方法 当filter对象销毁时执行该方法，关闭服务器时
    ③在web.xml中进行配置(配置要对哪些资源进行过滤)（filter-mapping在前面的先执行）
        <filter>
        <filter-name>EncodingFilter</filter-name>
        <filter-class>com.ithiema.web.filter.EncodingFilter</filter-class>
        </filter>
        <filter-mapping>
        <filter-name>EncodingFilter</filter-name>
        <url-pattern>/*</url-pattern>
        </filter-mapping>
        url-pattern配置①完全匹配 /sertvle1②目录匹配 /aaa/bbb/*③扩展名匹配*.abc  *.jsp
    注意：url-pattern可以使用servlet-name替代，也可以混用

作用

①公共代码的提取
    ②对request和response中的方法进行增强(装饰者模式/动态代理)
    ③进行权限控制

生命周期

①服务器启动时就创建该filter对象，每次访问被拦截目标资源，过滤器的doFilter方法就会执行
    ②服务器关闭时filter销毁
