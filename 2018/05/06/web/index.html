cookie 描述
创建Cookie Cookie cookie = new Cookie(name,value)
cookie会以响应头的形式发送给客户端：set-cookie:”name=zhengsan”
不能存储中文、它是浏览器缓存的一部分
数据存储在客户端本地，减少服务器端的存储的压力
安全性不好，客户端可以清除cookie,有大小和个数的限制
持久化时间 cookie.setMaxAge(秒)不设置cookie会存储在浏览器的内存中，随浏览器关闭销毁(会话级别cookie)
设置cookie信息会被持久化到浏览器的磁盘文件里，过期浏览器自动删除(持久级别cookie)
请求时cookie被携带路径 cookie.setPath();不设置，cookie信息会在访问产生该cookie的web资源所在的路径都携带cookie信息，会覆盖
cookie.setPath(“/WEB”);访问WEB应用中的任何资源都携带cookie
cookie.setPath(“/WEB/cookieServlet”);访问WEB16中的cookieServlet时才携带cookie信息
cookie.setPath(“/“);代表访问tomcat下所有的web项目的cookie信息
向客户端发送cookie response.addCookie(Cookie cookie);//如果路径和名称一样，两次add会覆盖
获得客户端的cookie Cookie[] cookies = request.getCookies();
获得cookie的键值 cookie.getName();cookie.getValue();
删除cookie Cookie cookie = new Cookie(“name”,””);//将cookie的name设置与删除的cookie一致
cookie.setPath(“/WEB16”);//将path设置成与要删除cookie的path一致
cookie.setMaxAge(0);//设置时间是0;response.addCookie(cookie);
session 描述
获得Session request.getSession([boolean create])
将数据存储到服务器端，安全性相对好，增加服务器的压力
session会为每个客户端都创建一块内存空间存储客户的数据
但客户端需要每次都携带一个标识ID去服务器中寻找属于自己的内存空间。
所以说Session的实现是基于Cookie，Session需要借助于Cookie存储客户的
唯一性标识JSESSIONID来辨别是哪个客户端
获得sessionid String getId()
获得新建session时与1970年1月1月0点时间差毫秒 long getCreationTime()
获得最后一次session时与1970年1月1月0点时间差毫秒 long getLastAccessedTime()
设置session空闲时间 void setMaxInactiveInterval(int interval)

WEB对象生命周期 创建 销毁 域的作用范围
servlet(多个) init第一次访问创建、service每次请求都会执行 destory服务器关闭、项目被移除
request(多个) 创建:访问时创建、域的作用范围:一次请求中 销毁：响应结束 一次请求中
ServletContext(只有一个) web应用被加载（服务器启动或发布web应用（前提，服务器启动状态）） web应用被卸载（服务器关闭，移除该web应用） 整个web应用
session 第一次执行getsession获得session、没有则创建 非正常关闭服务器（正常关闭session被序列化） 一次会话中
session过期，默认时间30分钟，调用invalidate方法
filer 服务器启动时 服务器关闭 每次访问被拦截目标资源，过滤器doFilter方法执行
