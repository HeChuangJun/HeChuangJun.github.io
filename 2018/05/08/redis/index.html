redis（nosql数据库）

redis

关系型数据库是基于关系表的数据库，最终会将数据持久化到磁盘上
    nosql数据库是基于特殊的结构，并将数据存储到内存的数据库
    从性能上而言，nosql数据库要优于关系型数据库
    从安全性上而言关系型数据库要优于nosql数据库
    一个项目中nosql和关系型数据库会一起使用，达到性能和安全性的双保证。

安装与启动

在redis的bin目录下可以执行./redis-server前端启动方式
    在bin下执行命令,/redis-server redis.conf(后端启动)
    关闭：关闭进程、在客户端下shutdown

常用命令

redis是一种高级的key-value的存储系统

key是字符串类型

key不要太长，最好不要操作1024个字节，这不仅会消耗内存还会降低查找效率
    key不要太短，如果太短会降低key的可读性
    在项目中，key最好有一个统一的命名规范（根据企业的需求）

Key 五种数据类型

字符串型 string

概述
基础的数据存储类型、在Redis中是二进制安全的
    该类型可以接受任何格式的数据，如JPEG图像数据或Json对象描述信息
    Redis中字符串类型的Value最多可以容纳的数据长度是512M
操作
set key value设置key的value该key存在则进行覆盖操作
    get key获得key的String值如果该key不存在，返回null.
    如果与该key关联的value不是String类型，报错
    getset key value先获取该key的值，然后在设置该key的值
    incr|decr key将指定的key的value原子性的递增|减1.
    不存在，初值0，在incr之后其值为1|-1（没设置输出1|-1）。
    如果value的值不能转成整型，如hello，该操作将执行失败并返回相应的错误信息
    incrby|decrby key increment|decrement将指定的key的value
    原子性增加|减小increment|decrement，
    key不存在，初值为0，在incrby之后，该值为increment|decrement。
    如果该值不能转成整型，如hello则失败并返回错误信息
    append key value如果该key存在，则在原有的value后追加该值；
    如果该key不存在，则重新创建一个key/value

字符串列表 lists

概述
在插入时，如果该键并不存在，Redis将为该键创建一个新的链表。
    与此相反，如果链表中所有的元素均被移除，那么该键也将会被从数据库中删除。
    List中可以包含的最大元素数量是4294967295。
    在链表的两头插入或删除元素(高效),即使链表中已经存储了百万条记录
    该操作也可以在常量时间内完成。元素插入或删除操作是作用于链表中间(低效)
操作
List类型是按照插入顺序排序的字符串链表、可以在其头部(left)和尾部(right)添加新的元素
    在list的头部添加多个values，不存在就创建。lpush key value1 value2..
    在list的尾部添加多个values，不存在就创建。rpush key value1、value2….
    仅key存在时在list的头部添加value lpushx key value
    仅key存在时在list的尾部添加value rpushx key value
    返回并弹出链表中的头部value。删除了 lpop key
    返回并弹出链表中的尾部value。删除了 rpop key
    在pivot元素前或者后插入value。linsert key before|after pivot value：
    返回指定链表中的value数量 llen key
    删除count个value，lrem key count value，count大于0，从头向尾遍历，count小于0，
    从尾向头遍历，count等于0，删除链表中所有value
    设置链表中的index的脚标的元素值，0代表链表的头元素，
    -1代表链表的尾元素lset key index value
    获取链表中从start到end的元素的值，start、end可正负，-1表示链表尾部元素，
    -2倒数第二个，lrange key 0 -1 查全部 lrange key start end
    将链表中的尾部元素弹出并添加到头部（两个集合的操作）rpoplpush key1 key2

字符串集合 sets

概述
相当于没排序的不重复的list
    操作的时间是常量时间。Set可包含的最大元素数是4294967295
    在服务器端完成多个Sets之间的聚合计算操作，如unions、intersections和differences。
    由于这些操作均在服务端完成，    因此效率极高，而且也节省了大量的网络IO开销
操作
向set中添加数据，如果该key的值已有则不会重复添加 sadd key value1、value2…
    删除set中指定的成员 srem key member1、member2…
    获取set中所有的成员 smembers key
    获取set中成员的数量 scard key
    判断参数中指定的成员是否在该set中，1存在，0不在或者key本身就不在sismember key member
    随机返回set中的一个成员srandmember key
    与key的顺序有关。返回差集”相减“sdiff key1 key2
    将key1、key2相差的成员存储在destination上sdiffstore destination key1 key2
    返回交集sinter key key1,key2…
    将返回的交集存储在destination上sinterstore destination key1 key2
    返回并集。sunion key1、key2
    将返回的并集存储在destination上sunionstore destination key1 key2

有序字符串集合 sorted sets

概述
游戏排名、微博热点话题等使用场景。
    Sorted-Sets中的每一个成员都会有一个分数(score)与之关联的set
    Redis正是通过分数来为集合中的成员进行从小到大的排序。
    然而需要额外指出的是，Sorted-Sets中的成员必须是唯一的，分数(score)可以重复
    在Sorted-Set中添加、删除或更新一个成员都是非常快速的操作，
    其时间复杂度为集合中成员数量的对数。由于Sorted-Sets中的成员在集合中的位置是有序的，
    因此，即便是访问位于集合中部的成员也仍然是非常高效的
操作
将所有成员以及该成员的分数存放到sorted-set中zadd key score member score2 member2 …
    移除集合中指定的成员，可以指定多个成员。zrem key member[member…]
    设置指定成员的增加的分数increment。zincrby key increment member
    获取集合中的成员数量zcard key
    获取分数在[min,max]之间的成员zcount key min max
    获取集合中脚标为start-end的成员及分数zrange key start end [withscores]
    返回分数在[min,max]的成员并按照分数从低到高排序。[limit offset count]：
    offset，表明从脚标为offset的元素开始并返回count个成员。
    zrangebyscore key min max [withscores] [limit offset count]
    返回成员在集合中的位置 zrank key member
    返回指定成员的分数 zscore key member

哈希类型 hashs

概述
具有String Key和String Value的map容器、非常适合于存储值对象的信息
    如果Hash中包含很少的字段，那么该类型的数据也将仅占用很少的磁盘空间。
    每一个Hash可以存储4294967295个键值对。
操作
为指定的key设定field/value对（键值对） hset key field value
    设置key中的多个filed/value hmset key field value
    返回指定的key中的field的值hget key field
    获取key中的多个filed的值hmget key field value
    获取key中的所有filed-vaule hgetall key
    判断指定的key中的filed是否存在hexists key field value
    获取key所包含的field的数量hlen key
    设置key中filed的值增加increment hincrby key field increment
