Java常用API

API：Application(应用) Programming(程序) Interface(接口)

Java API就是JDK中提供给我们使用的类，这些类将底层的代码实现封装了起来，

Object类：所有类的父类（超类）

boolean equals（Object obj）

①比较两个对象是否相同（比较地址，方法内部使用的就是==比较运算符）
    ②开发经常需要子类重写equals方法根据对象的属性值进行比较。
    ③复写equals方法时，一定要注意public boolean equals(Object obj)的参数是Object类型，
    在调用对象的属性时，一定要进行类型转换，在转换之前必须进行类型判断instanceof
/*
 描述人这个类，并定义功能根据年龄判断是否是同龄人
 由于要根据指定类的属性进行比较，这时只要覆盖Object中的equals方法
 在方法体中根据类的属性值进行比较
*/
    class Person extends Object{
        int age ;
        //复写父类的equals方法，实现自己的比较方式
        public boolean equals(Object obj) {
            //判断当前调用equals方法的对象和传递进来的对象是否是同一个
            if(this == obj){
                return true;
            }
            //判断传递进来的对象是否是Person类型
            if(!(obj instanceof Person)){
                return false;
            }
            //将obj向下转型为Perosn引用，访问其属性
            Person p = (Person)obj;
            return this.age == p.age;
        }
    }

String toString（）返回该对象的字符串(包名.类名+@+内存地址值)

经常需要按照对象的属性得到相应的字符串表现形式，因此也需要重写它

String类：引用型变量

字符串是常量；它们的值在创建之后不能更改，内存区域中就生成了这个字符串。
    字符串的值本身不能改变，一旦没有引用就会被垃圾回收机制回收
    但引用字符串的变量中记录的地址值是可以改变的

构造

String s3 = "abc";//s3创建，在内存中只有一个对象。这个对象在字符串常量池中
    String s4 = new String("abc");//s4创建，在内存中有两个对象。new的对象在堆中，
    字符串本身对象在字符串常量池中
    s3==s4;//false，比较的是内存地址
    s3.equals(s4);//true,因为String重写了equals方法，通过字符串对象中的字符来判断
    String() 初始化一个新创建的 String 对象，使其表示一个空字符序列
    String(String original) 以参数为模板初始化一个String对象
    String(byte[] bytes) 使用平台的默认字符集解码指定的byte数组,构造一个新String
    String(byte[] bytes, int offset, int length) 默认字符集解码byte子数组构造新String
    String(char[] value) 分配一个新的String，使其表示字符数组参数中当前包含的字符序列
    String(char[] value, int offset, int count) 新String取自字符数组参数子数组的字符

    byte[] bys = new byte[]{97,98,99,100};
    String s2 = new String(bys); //创建String对象，把数组元素作为字符串的内容
    String s3 = new String(bys, 1, 3); //创建String对象，把部分数组元素作为字符串的内容
    参数offset为数组元素的起始索引位置，参数length为要几个元素
    char[] chs = new char[]{’a’,’b’,’c’,’d’,’e’};
    String s4 = new String(chs); //创建String对象，把数组元素作为字符串的内容
    String s5 = new String(chs, 0, 3);//创建String对象，把部分数组元素作为字符串的内容
    参数offset为数组元素的起始索引位置，参数count为要几个元素

StringBuffer类：字符缓冲区（支持可变的字符串）

字符串的缓冲区是一个容器，容器中可以装很多字符串。并且能够对其中的字符串进行各种操作
    在原有字符串缓冲区内容基础上，在末尾追加新数据StringBuffer append(String str) 
    在指定位置插入新数据StringBuffer insert(int offset,String str)
    删除指定位置上的字符StringBuffer deleteCharAt(int index) 
    删除指定范围内的多个字符StringBuffer delete(int start,int end) 
    将指定范围内多个字符用新字符替换StringBuffer replace(int start,int end,String str)
    将字符串缓冲区的内容 反转  "abc"----"cba"StringBuffer reverse()

StringBuilder类可变的字符序列，不同步，但是速度比StringBuffer 快

    注意事项

        在Calendar类中，月份的表示是以0-11代表1-12月

DateFormat：与语言环境无关的方式格式化并解析日期或时间的抽象类

子类SimpleDateFormat：字符串与Date对象相互转换

SimpleDateFormat() 默认的格式化操作
    SimpleDateFormat(String pattern)创建格式Date模板,如”yyyy-MM-dd-HH-mm-ss-SS”    
    规则是在一个字符串中，会将以上字母替换成对应时间组成部分，剩余内容原样输出：
    final String format(Date date) 把日期格式化成字符串
    Date date = new Date(1607616000000L);//Fri Dec 11 00:00:00 CST 2020
    DateFormat df = new SimpleDateFormat(“yyyy年MM月dd日”);
    String str = df.format(date);//str中的内容为2020年12月11日
    Date parse(String source) 把日期字符串转换成日期对象
    String str = ”2020年12月11日”;
    DateFormat df = new SimpleDateFormat(“yyyy年MM月dd日”);
    Date date = df.parse( str );//Date对象Fri Dec 11 00:00:00 CST 2020
