    node.js实现了javascript在服务器端运行，方便前端工程师与后端工程是协同开发

    node.js介绍

    官网：https://nodejs.org/
    Node.js® is a JavaScript runtime built on Chrome's V8 JavaScript engine.
    node.js可以像浏览器一样解析JavaScript，是JavaScript在服务器端的运行环境    
    node.js只使用ecmascript,而不使用bom和dom，因为node.js不操作界面
    node.js还有一个强大的工具npm，管理了所有与Javascrpit相关的包如jquery等

    javascript在浏览器的引擎和node.js运行上的区别

    浏览器：能进行bom和dom操作，不能操作文件
    node.js：操作文件、运行服务等，但不能使用bom和dom

    node.js的功能

    1、web应用服务器
    2、命令行工具（git（c）、hexo（node）、npm（node））

    node.js和JavaScript的异同

    node包括ecmascript、第三方模块（npm下载）、核心模块、用户自定义模块;
    javascript包括ecmascript、bom和dom;

    Node中的Console（用于只读测试，不用谢require）

    在cmd输入node回车进入console，按ctrl+c两次退出node环境

    node.js安装

    使用msi安装：然后在cmd中输入node -v 有显示版本号说明安装成功

    helloworld案例

    在cmd或者gitBash或者windows的powershell下定位到

    相对于执行node的路径未为根路径/（相对路径./data/a于data/a/一样）
    在加载模块的时候不能省略./，剩下的与模块标识一致

    node中的其他成员

    在每个模块中，除了require、exports等模块相关api中，还有两个特殊的成员
    _ _dirname可以永联获取当前文件模块所属目录的绝对路径（目录）
    _ _filename可以获得当前文件的绝对路径（路径不包含文件）
    以上两个命令不受执行node命令的位置所影响经常配合path.join合用
    在node中，操作文件路径是相对执行node命令的路径(./)，所以用上面的比较可靠
    在node中，模块中的路径标识是相对于当前文件路径的，不受执行node命令的位置影响

    模块系统

    node.js三大模块：具名核心模块、第三方模块、用户自定义模块 模块作用域，使用require方法加载模块(加载了不一定能使用该模块的成员，必须导出了才能使用)使用exports接口对象导出模块中的成员

    node.js的模块系统的使用

    require方法的作用：

    模块作用域:默认文件中所有的成员只在当前模块有效
    1.加载文件模块并执行里面的代码（同时执行两个js文件）
    加载执行模块代码:require('./b.js');
    使用用户自定义模块，相对路径必须加./，且.js可以省略        
    node中没有全局作用域，只有模块作用域
    大意：只使用require有关系的两个js文件不能相互访问变量和函数只有exports出来了才能用
    2.拿到被加载文件模块导出的接口对象export,test就是exports
    需要在被加载文件中添加exports.add='hello'
    在需要加载文件中var test=require('./b.js');test.add
    在每个模块中都提供了一个对象：exports（默认为{}）
    把需要被外部访问的成员放到export中即可破除原来的模块作用域限制

    模块系统底层

    require模块加载规则

    1优先从缓存中加载，加载b，b加载c，且a加载c，那么c不会被两次执行，
    此时a只获取c中的接口对象，并不能执行例如打印字段的代码
    2判断模块标识require(‘模块标识’);核心模块、用户自定义模块、第三方模块
    路径形式:./或者../或者/(当前文件模块所在的磁盘根路径：少用)不能省
    核心形式:'http'等，已经被编译到二进制文件中了，只需按名字加载】
    3第三方模块:所有的第三方模块都需要npm下载，才能被加载
    先找当前文件所处目录中的node_modules目录
    node_modules/模块名
    node_modules/模块名/package.json文件
    node_modules/模块名/package.json文件的main属性（默认为index.js）
    main属性记录了第三方模块的入口模块，然后加载使用第三方包
    实际上最终加载的还是文件，如果package.json或者入口模块不在，则报错
    node.js在以上任何一个条件不符合的情况下，会一直向上查询直到磁盘根目录
    我们项目根目录只需一个node_modules，不能加载同级或者子目录下的模块（手动加载可以解决），故放根目录合适

    包说明package.json文件：（不锁定版本，自动升级到最新版本）

    每个项目都应该有package.json说明依赖等信息
    初始化项目（在需要建立项目的目录下cmd）命令:npm init
    然后按照提示完成初始化就会在项目根目录下自动生成package.json文件
    (npm install --save就能保存项目的依赖说明)    
    只要package.json文件的“dependencies”存在，此时执行npm install即可找回以前的依赖

    packag-lock.json文件

    保存了所有的依赖信息（依赖版本、下载地址等）到时执行npm install 会比没有这个文件速度快很多，且只会锁定版本

    npm（node package manager）常用命令

    npm官网和npm命令行工具，只要安装了node就自带，npm --version,只有在他官网上能搜索到的才能install

    moment组件用于日期格式化

    npm i moment -S import moment from ‘moment’ moment(需要被格式化的字符串).format(格式化后的字符串格式)

    解决npm被墙问题：速度慢

    http://npm.taobao.org/将npm备份到国内
    安装npm install --global cnpm安装包的时候讲命令改成cnpm
    --global表示安装到全局，非当前目录（以上命令在任何目录执行都可以）
    不安装cnpm又想使用淘宝仓库，则npm install jquery --registry=https://registry.npm.taobao.org/
    去掉每次都写的麻烦，则npm config set registry https://registry.npm.taobao.org/

    解决服务器改代码重启服务器麻烦

    npm intall --global nodemon
    以后执行的时候就执行nodemon xxx.js，热部署
