所有api详解（视图修改会导致vm中的m修改！！！）

ko.applyBindings(?,[?]);

第一个参数vm对象（必须），第二个参数document元素（可选，限制使用范围）
    例如ko.applyBindings(vm,document.getElementById('someElementId'));

“单向绑定”(使用applyBindings绑定后修改m不会导致v变化)

“双向绑定”(使用applyBindings绑定后修改m导致v变化)

强制刷新（只要m中的值被修改）全部支持

vm.personName.extend({ notify: 'always' });

延时加载（当刷新比较频繁的时候）全部支持

vm.personName.extend({ rateLimit: 50 });//50毫秒后刷新

双向绑定数组（支持延时加载）

（对于数组来说实现双向绑定必须vm.数组名.数组方法 来修改数组）
    （而不是vm.数组名().数组方法 修改数组，虽然两种方法都能修改数组，
    但是只有第一种方法满足我们的要求，因为第一种是js原生方法）

计算方法ko.computed(function(){})（一个data-bind监控多个data-bind的函数）

只要其中一个data-bind发生变化，计算方法中的data-bind所在的view就会显示

ko.purecomputed(function(){})，

这是一个更加高效的方法，用于替代不涉及修改其他对象的computed方法
    防止内存泄漏和减少计算开销来提高性能

ko.computed能通过判断来决定那个组件显示

去除‘this’

all“bind”(0\null\undefined为false)

data-bind=”visible:function(){}”通过优先将display置于none或者block等来将组件可视化

data-bind=”text: function(){}”//绑定string，如果是不合法的用toString()

data-bind=”html: details”//底层是用innerhtml写的

data-bind=”css: { profitWarning: xxx() < 0 }”、data-bind=”css: { ‘my-class’: xxx}”>

data-bind=”style:{color:xxx()<0?’red’:’black’}”、data-bind=”style:{‘my-class’:xxx}”

data-bind=”attr: {href: url,title: details}”、data-bind=”attr:{‘my-class’:xxx}”

data-bind=”foreach: people”//迭代数组

foreach动画data-bind=”foreach: { data: myItems, afterAdd: yellowFadeIn }”

可以用的函数有afterRender/afterAdd/beforeRemove/beforeMove/afterMove

特殊遍历+判断+with

data-bind=”if: displayMessage”

data-bind=”with: coords”

以下事件机制与第一种大概相同

data-bind=”click: incrementClickCounter”,data-bind=”click: xxx.xxx”
